# 소비 · 저축 어드바이저

마이데이터(자체 더미 제공자 서버) 기반의 소비 분석 · 금융상품 추천 · 이상거래 탐지(FDS) 학습용 포트폴리오 MVP.

> 결정·설계 근거의 단일 기준은 마스터 문서 `reference/finntech_things.md`다. 이 README는 실행·개요만 담고, 상세는 그 문서의 §번호로 인용한다.

## 아키텍처 한눈에

```
frontend (5173)  ──▶  backend (8080)  ──▶  backend-mydata (8082, 마이데이터 제공자)
                          │
                          └─ 가상 본인인증(CI) → 마이데이터 연동(카드·결제 적재)
                             → Consumption(source=MYDATA) 투영 → 기존 분석 엔진 재사용
```

- **backend** — 분석 엔진(소비건전성·추천·FDS)·감사로그·개인정보·게임화 저축·통장 비교.
- **backend-mydata** — 은행/카드사 측 더미 데이터를 제공하는 별도 프로세스(마이데이터 시뮬레이션). 개인은 실 마이데이터에 접근할 수 없어(마스터 §III-B) 공개 표준 스펙 모양의 더미를 우리가 운영한다.
- **frontend** — React + Vite. 온보딩(본인인증→금융사 연동) 후 홈·내 카드·내 소비 등.

## 빠른 실행

```bash
./scripts/dev-up.sh          # backend + backend-mydata 빌드·기동·시드 (H2, Docker 불필요)
cd frontend && npm run dev   # http://localhost:5173
```

TSA 계층3까지: `TSA_ENABLED=true ./scripts/dev-up.sh`.

## 주요 기능

- **마이데이터 연동** — 가상 본인인증(CI)으로 마이데이터 서버에서 카드·소비내역을 불러와 적재(§13).
- **소비건전성지수** — 저축진행률·안정성·계획소비로 A~D 등급(§5).
- **이상거래 탐지(FDS)** — Modified Z-score(median·MAD) AND 룰 결합(§5).
- **감사로그 3계층** — 해시체인 + Merkle + RFC 3161 TSA 앵커링(내부자 재작성까지 탐지, §5-4).
- **게임화 저축** — 가상 포인트·목표 버킷·충동예산 절약통·고민 목록(§5-5).
- **통장 비교(정보성)** — 실 예·적금 금리 비교(무판매·무제휴, 금융위 유권해석 근거, §5-5).

## 설계 원칙 (마스터 §4)

1. **판단은 코드가, 표현은 AI가** — 엔진이 계산한 수치를 AI는 문장으로만 옮긴다(설명가능성·개인정보).
2. **하나의 분석 결과를 세 갈래로 재사용** — 서비스는 임계치를 재계산하지 않는다.
3. **재현성** — 엔진은 `now()`를 직접 읽지 않고 `Clock`·`referenceTime` 주입, 조회 정렬 고정, `TreeMap`.
4. **세그먼트 비의존** — 카테고리를 코드에 박지 않고 임계치는 전부 `application.yml`.

## 규제 안전 (요약)

추천/판매 흐름의 금융상품은 **전부 더미**라 실계약·수수료가 없어 금소법 '영업'이 아니다. 단 **통장 비교(정보성)**는 무판매·무제휴·가입편의 없음 조건에서 실 금리를 표시한다(금융위·금감원 법령해석 2022.6.15). 상세는 마스터 §III-B.
