## 0. 근거 표시 기준

| 표시 | 의미 |
|---|---|
| `근거 있음` | 내부 리서치, 공식 통계, 경쟁사 공식 기능 또는 실제 사용자 반응에서 확인 |
| `기획 확정` | PRD상 만들기로 결정했으나 고객 효과는 아직 검증되지 않음 |
| `가설` | 인터뷰·프로토타입·출시 데이터로 확인해야 하는 판단 |
| `미확인` | 현재 문서에 판단할 근거가 없음. 임의로 채우지 않음 |

`근거 있음`도 시장 전체를 대표한다는 뜻은 아니다. 현재 문제 근거는 문헌과 온라인 발화 중심의 정성 탐색이며, 실제 타깃 고객 인터뷰와 정량 검증은 아직 완료되지 않았다.

## 2. 블록별 상세

### ① 문제

#### 해결하려는 상위 문제 3개

1. 스트레스와 보상 심리로 충동 소비를 반복하고 뒤늦게 후회한다.
2. 소비 내역을 보더라도 무엇부터 줄여야 할지 판단하지 못한다.
3. 이번 달 절약이 얼마나 진행됐는지 월말 전에 체감하지 못해 행동을 지속하기 어렵다.

#### 보조 문제

- 전체 소비를 평가하거나 반복 경고하는 서비스는 잔소리·감시로 느껴진다.
- 자동 분류가 틀리거나 수정하기 어려우면 서비스 전체를 믿지 않게 된다.
- 월 단위·카테고리 중심 예산은 변동 지출을 충분히 설명하지 못할 수 있다.

#### 기존 대안

| 대안 | 현재 확인된 역할 | 남는 공백 |
|---|---|---|
| 토스·뱅크샐러드 등 자산관리 | 거래와 카테고리별 소비 확인 | 분석이 다음 행동 기준으로 이어지는지는 미확인 |
| Rocket Money·PocketGuard 등 예산 앱 | 자동 연동, 월 예산, 가용 금액 계산 | 동기화 신뢰, 카테고리 복잡성, 고정 월 예산 불만이 반복 관찰됨 |
| Cleo 등 AI 재무 코칭 | 대화형 조언과 행동 개입 | 구독·현금 선지급 이해관계가 조언 신뢰를 해칠 수 있음 |
| 엑셀·구글시트 | 사용자가 직접 기준과 계산을 통제 | 입력·유지 부담, 신규 거래 자동 반영 없음 |
| 카카오톡·메모·종이 가계부 | 빠른 자기 기록 | 자동 집계·상태 계산·월간 지속 구조 없음 |
| 별도 통장 이체 | 지킬 돈을 실제로 분리 | 시작 부담과 중도 수정 부담이 큼 |

#### 근거 상태

- `근거 있음`: 문헌과 온라인 발화에서 충동 소비, 늦은 인지, 절감 기준 부족, 중간 성과 부재가 반복 확인됐다.
- `근거 있음`: 최근 경쟁사 사용자 반응에서 연동 지연·누락·중복·경직된 월 예산 문제가 관찰됐다.
- `미확인`: 실제 20–30대 직장인 중 각 문제가 발생하는 비율.
- `미확인`: 세 문제 중 어느 하나가 실제 결제·지속 사용을 만드는 최우선 문제인지.

### ② 고객군

#### 초기 고객 가설

> 스트레스 때문에 배달·쇼핑·택시를 이용한 뒤 후회하지만, 무엇을 줄여야 할지 몰라 절약을 오래 이어가지 못하는 20–30대 직장인

#### 초기 수용자 가설

다음 조건을 모두 또는 대부분 가진 25–39세 직장인을 첫 검증 대상으로 본다.

1. 최근 3개월 안에 충동 소비 후 후회한 경험이 있다.
2. 가계부나 절약 앱을 사용했지만 현재는 적극적으로 쓰지 않는다.
3. 절약 시도를 한 달 이상 유지하기 어려웠다.
4. 줄일 수 있는 소비를 찾고 싶지만 무엇부터 줄일지 모르거나 지속하지 못한다.

#### 고객이 원하는 진전

> 내가 지키기로 한 돈이 지금 얼마나 보호되고 있는지 확인하면서, 반드시 지키고 싶은 소비는 건드리지 않고 소비를 조정하고 싶다.

#### 시장 범위 참고

| 범위 | 규모 | 근거 상태 |
|---|---:|---|
| 연령 기준 TAM | 약 1,226만 명 | 2026년 6월 국내 20–39세 주민등록 인구에 근거 |
| 조건 적용 SAM | 약 119만 명 | 취업자 수에 임금근로자 비율과 문제 적합률을 적용한 추정 |
| 현재 프로토타입 SOM | 참여 100–300명·월간 완료 30–90명 | 모집·완료율을 가정한 운영 추정 |

#### 근거 상태

- `근거 있음`: 해당 행동과 감정 패턴이 정성 리서치에서 관찰됐다.
- `가설`: 20–30대 직장인이 가장 반응이 높은 초기 세그먼트다.
- `가설`: 위 네 조건을 가진 사람이 초기 수용자가 된다.
- `미확인`: 금융 데이터 연결 의향, 반복 사용 의향, 결제 의향.
- `미확인`: 페르소나의 소득·카드 사용액·앱 이용 행태가 실제 고객군을 대표하는지.

### ③ 고유 가치 제안

#### 제안 문장

> **줄일 수 있는 소비만 골라, 지킨 돈을 한 숫자와 한 캐릭터로 이어간다.**

#### 풀어쓴 문장

> MOA는 모든 지출을 통제하는 가계부가 아니라, 내가 줄이기로 한 소비에서 아직 지켜지고 있는 돈을 한눈에 보여주고 그 돈을 캐릭터의 여정으로 이어주는 절약 동반자다.

#### 가치의 구성

- `선택권`: AI는 후보와 이유만 제안하고 사용자가 카테고리와 강도를 정한다.
- `단순성`: 홈은 내부 사용 한도를 숨기고 현재 지키는 금액 하나를 보여준다.
- `부분 절감`: 카테고리 전체 금지 대신 사용 한도를 넘긴 금액만 차감한다.
- `비난 없음`: 지출을 실패로 만들거나 캐릭터를 퇴화시키지 않는다.
- `동기 치환`: 물건·여행 목표 없이 월말 확정 지킨 돈을 캐릭터 성장으로 바꾼다.

#### 근거 상태

- `기획 확정`: 위 가치 요소와 금지 원칙은 PRD에 반영됐다.
- `가설`: 사용자가 `지킨 돈`을 잔액·남은 예산·저축액과 구분해 이해한다.
- `가설`: 한 숫자가 기존 카테고리 예산보다 더 간단하고 행동에 도움이 된다.
- `가설`: 외부 소비 목표 없는 캐릭터 성장이 재방문 동기가 된다.
- `미확인`: 어떤 문장이 실제 고객의 가장 높은 관심·가입·재방문을 만든다.

### ④ 해결책

#### 1차 MVP 흐름

```text
더미 마이데이터로 과거 3개월 소비 분석
→ AI가 줄여볼 카테고리와 근거 제안
→ 사용자가 1–2개 카테고리와 절약 강도 선택
→ 이번 달 지킬 돈 확정
→ 신규 거래 시뮬레이션과 선택 카테고리 자동 반영
→ 사용 한도 이내에서는 침묵
→ 한도 초과분만 지킬 돈에서 차감
→ 오분류·취소·환불 시 재계산
→ 월말 최종 지킨 돈 확정
→ 캐릭터 누적 성장과 새 챕터
```

#### 해결책의 핵심 규칙

```text
사용 한도 = 기준 소비 × (1 − 절약 강도)
지킬 돈   = 기준 소비 × 절약 강도
지킨 돈   = max(지킬 돈 − max(누적 소비 − 사용 한도, 0), 0)
```

#### 현재 구현 경계

- 실제 개인신용정보와 실제 자금은 다루지 않는다.
- 실제 마이데이터가 아니라 더미 서버와 신규 거래 시뮬레이션을 사용한다.
- 실제 계좌 개설·이체, 푸시 알림, OS 위젯은 1차 MVP 범위가 아니다.
- 금융상품은 정보성 비교만 제공하며 판매·중개·가입 대행을 하지 않는다.

#### 근거 상태

- `기획 확정`: 화면 흐름, 계산 규칙, 캐릭터 성장 규칙.
- `가설`: 절약 후보와 근거가 카테고리 선택을 쉽게 만든다.
- `가설`: 초과분 차감이 다음 소비 행동을 조정한다.
- `가설`: 사용자 수정 기능이 자동 분석 신뢰를 회복시킨다.
- `가설`: 월말 성장과 새 챕터가 다음 달 재사용을 만든다.
- `미확인`: 실제 마이데이터 환경에서도 동일한 정확도·속도·신뢰를 제공할 수 있는지.

### ⑤ 채널 — 미확인

현재 문서에는 다음을 확정할 근거가 없다.

- 첫 고객을 어디에서 지속적으로 획득할지
- 앱스토어 검색, 콘텐츠, 커뮤니티, 제휴 중 어느 채널이 유효한지
- 금융사·카드사·학교·기업과 제휴할지
- 채널별 가입 전환율과 고객획득비용(CAC)
- 유료·무료 획득 비중

[시장·경쟁 분석](./11_시장경쟁분석.md)의 SOM 계산에서 `팀 직접 모집·제한적 공개 테스트`를 사용했지만 이는 시장 규모를 계산하기 위한 가정일 뿐, 검증된 획득 채널이 아니다.

> **현재 판정: 미확인**

---

### ⑥ 수익원 — 미확인

현재 확인되지 않은 항목:

- 누가 돈을 내는지
- 구독, 일회성 구매, B2B, 제휴 등 어떤 방식으로 수익을 낼지
- 무료와 유료 기능의 경계
- 가격과 결제 의향
- 유료 전환율과 해지율
- 금융상품 정보성 비교가 수익과 연결되는지

현재 금융상품 기능은 판매·중개·가입 대행을 하지 않는 정보성 비교다. 따라서 이를 제휴 수수료나 중개 매출로 계산할 근거가 없다.

가격이 없으므로 금액 기준 TAM·SAM·SOM과 매출 전망도 만들 수 없다.

> **현재 판정: 미확인**

- 캐릭터 콘텐츠 제작·운영 비용
- 마케팅비와 고객획득비용
- 고객지원·환불·민원 처리 비용
- 고정비와 사용자당 변동비

> **현재 판정: 비용 발생 항목은 부분 확인, 금액과 구조는 미확인**

### ⑧ 핵심 지표

#### 현재 PRD의 측정 후보

| 단계 | 지표 | 현재 상태 |
|---|---|---|
| 진입 | 온보딩 완주율 | 정의 있음·실측값 미확인 |
| 선택 | 카테고리 선택 완료율 | 정의 있음·실측값 미확인 |
| 추천 | AI 추천 카테고리·권장 강도 수용률 | 정의 있음·실측값 미확인 |
| 이해 | 지킬 돈 의미 이해·차감 이유 설명 가능 | 목표 80%·근거와 실측값 미확인 |
| 신뢰 | 거래 자동 반영 성공률·오분류 수정률 | 정의 있음·실측값 미확인 |
| 행동 | 같은 카테고리의 다음 소비 변화 | 측정 제안·실측값 미확인 |
| 월말 | 월말 성장 확인률 | 정의 있음·실측값 미확인 |
| 지속 | 다음 달 챕터 시작률·2개월 누적 성장 유지율 | 정의 있음·측정 방식 일부 미정 |
| 감정 | 비난·감시·강압으로 느낀 비율 | 측정 제안·실측값 미확인 |

#### 아직 정하지 못한 것

- 하나의 북극성 지표
- 제품 사용이 실제 절약 행동을 만들었다고 판단할 최소 효과 크기
- 더미 데이터에서 측정한 행동을 실제 금융 데이터 환경에 일반화할 기준
- 리텐션 측정 기간과 코호트 기준
- 수익·결제 관련 지표

`평균 지킨 돈`만 핵심 지표로 쓰지 않는다. 필요한 소비까지 줄인 결과를 성공으로 오판할 수 있으므로 통제감, 이해, 신뢰, 지속을 함께 본다.

> **현재 판정: 측정 후보는 기획 확정, 우선순위·기준값·실측값은 미확인**

### ⑨ 불공정 우위 — 미확인

린 캔버스의 불공정 우위는 경쟁사가 돈이나 인력을 투입해도 쉽게 복제하기 어려운 자산을 뜻한다.

현재 확인된 MOA의 차별점은 다음과 같다.

- 한도 초과분만 차감하는 부분 절감 공식
- 사용자가 선택한 소비만 반영하는 구조
- 현재 지키는 금액 한 숫자
- 비난·퇴화 없는 누적 캐릭터 성장

그러나 이들은 현재 공개된 기획과 UX 규칙이며 다른 서비스가 복제할 수 있다. 캐릭터 자체도 이미 경쟁 서비스에서 유사 표현이 존재하므로 방어 가능한 우위로 볼 근거가 없다.

아직 확인되지 않은 항목:

- 독점적이거나 장기간 축적된 데이터
- 경쟁사가 복제하기 어려운 학습 효과
- 독점 파트너십·유통망·라이선스
- 강한 커뮤니티·브랜드 충성도
- 특허·독점 기술
- 지속적으로 우수하다고 검증된 행동 변화 성과

> **현재 판정: 미확인**

## 3. 가장 위험한 미확인 가설

| 우선순위 | 미확인 가설 | 틀렸을 때 영향 | 확인 방법 |
|---|---|---|---|
| P0 | 타깃 고객이 `무엇을 줄일지 모름`과 `중간 성과 부재`를 실제 주요 문제로 느낀다 | 문제–고객 적합성 붕괴 | 25–39세 조건부 실제 인터뷰 |
| P0 | `지킬 돈`을 잔액·예산·저축액과 혼동하지 않는다 | 핵심 가치 이해 실패 | 계산 예시가 포함된 프로토타입 테스트 |
| P0 | 자동 분석이 틀려도 수정 후 신뢰가 회복된다 | 한 숫자 전체에 대한 불신 | 오분류·누락·중복 시나리오 테스트 |

| P1 | 사용자가 금융 데이터를 연결할 의향이 있다 | 실제 출시 활성화 실패 | 연결 의향·이탈 이유 인터뷰 |
| P1 | 다음 달 챕터를 다시 시작한다 | 월간 제품의 리텐션 실패 | 한 달 이상 종단 테스트 |

| P2 | 누군가 이 서비스에 돈을 낼 의향이 있다 | 사업 지속 불가 | **미확인 — 가격·지불의향 조사 필요** |
| P2 | 복제하기 어려운 우위를 축적할 수 있다 | 경쟁 방어력 부재 | **미확인 — 현 단계 검증 방법 미정** |

## 4. 현재 캔버스로 말할 수 있는 것과 없는 것

### 말할 수 있는 것

- MOA가 해결하려는 문제 패턴은 정성 리서치에서 반복 관찰됐다.
- 사용자가 선택한 소비의 지킬 돈을 부분 절감 방식으로 추적하는 제품 구조가 설계됐다.
- 기존 예산 앱에서 자동 연동 신뢰와 복잡한 월 예산 문제가 반복 관찰됐다.
- 현재 MVP는 실제 금융 서비스가 아니라 더미 데이터를 이용한 포트폴리오 프로토타입이다.

### 아직 말할 수 없는 것

- 타깃 고객이 MOA를 실제로 필요로 하고 반복 사용할 것이다.
- MOA가 충동구매를 사전에 막는다.
- 캐릭터가 리텐션을 높인다.
- AI 분석이 경쟁사보다 정확하다.
- 1년 안에 특정 가입자·매출을 달성한다.
- 검증된 유입 채널이나 수익모델이 있다.
- MOA에 복제하기 어려운 불공정 우위가 있다.

## 5. 캔버스 결론

현재 MOA는 `문제–고객–해결책`에 대한 구체적인 가설과 이를 시험할 MVP 설계까지는 갖춰져 있다.

반면 `채널–수익원–불공정 우위`는 근거가 없어 미확인 상태다. 비용 구조와 핵심 지표도 항목만 정의됐을 뿐 실제 값은 없다.

따라서 현 단계의 가장 중요한 과제는 사업 모델을 그럴듯하게 채우는 것이 아니라 다음 두 가지를 먼저 확인하는 것이다.

1. 타깃 고객이 `지킬 돈 한 숫자 + 부분 절감 + 비난 없는 성장`을 이해하고 실제 다음 소비를 조정하는가.
2. 한 달 뒤에도 다시 챕터를 시작할 만큼 가치가 남는가.

이 두 가설이 확인되기 전까지 채널 효율, 유료화, 매출, 불공정 우위를 확정 사실처럼 사용하지 않는다.
