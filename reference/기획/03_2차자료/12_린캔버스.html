지킨 돈,
한 숫자로. 줄일 수 있는 소비만 골라, 지킨 돈을 한 숫자와 한 캐릭터로 이어가는 절약 동반자

이 캔버스는 사업 모델을 채우기 위한 문서가 아니라, 무엇이 확인됐고 무엇이 아직 비어 있는지 구분하기 위한 원장입니다. 근거가 없는 칸은 그럴듯하게 채우지 않고 비워 뒀습니다.

근거 있음 내부 리서치, 공식 통계, 경쟁사 공식 기능 또는 실제 사용자 반응에서 확인
기획 확정 PRD상 만들기로 결정했으나 고객 효과는 아직 검증되지 않음
가설 인터뷰·프로토타입·출시 데이터로 확인해야 하는 판단
미확인 현재 문서에 판단할 근거가 없음. 임의로 채우지 않음

근거 있음도 시장 전체를 대표한다는 뜻은 아닙니다. 현재 문제 근거는 문헌과 온라인 발화 중심의 정성 탐색이며, 실제 타깃 고객 인터뷰와 정량 검증은 아직 완료되지 않았습니다.

해결책

설계 확정 · 효과 미확인

1차 MVP 흐름

더미 마이데이터로 과거 3개월 소비 분석
  ↓ AI가 줄여볼 카테고리와 근거 제안
  ↓ 사용자가 1–2개 카테고리와 강도 선택
  ↓ 이번 달 지킬 돈 확정
  ↓ 신규 거래 시뮬레이션 자동 반영
  ↓ 사용 한도 이내에서는 침묵
  ↓ 한도 초과분만 지킬 돈에서 차감
  ↓ 오분류·취소·환불 시 재계산
  ↓ 월말 최종 지킨 돈 확정
  ↓ 캐릭터 누적 성장과 새 챕터

계산 규칙

사용 한도 = 기준 소비 × (1 − 절약 강도)
지킬 돈  = 기준 소비 × 절약 강도
지킨 돈  = max( 지킬 돈 − max(누적 소비 − 사용 한도, 0), 0 )

현재 구현 경계

  • 실제 개인신용정보와 실제 자금은 다루지 않음
  • 더미 서버와 신규 거래 시뮬레이션 사용
  • 계좌 개설·이체, 푸시 알림, OS 위젯은 범위 밖
  • 금융상품은 정보성 비교만, 판매·중개·가입 대행 없음

현재 캔버스로 말할 수 있는 것과 없는 것

말할 수 있는 것

  1. MOA가 해결하려는 문제 패턴은 정성 리서치에서 반복 관찰됐다.
  2. 사용자가 선택한 소비의 지킬 돈을 부분 절감 방식으로 추적하는 제품 구조가 설계됐다.
  3. 기존 예산 앱에서 자동 연동 신뢰와 복잡한 월 예산 문제가 반복 관찰됐다.
  4. 현재 MVP는 실제 금융 서비스가 아니라 더미 데이터를 이용한 포트폴리오 프로토타입이다.

아직 말할 수 없는 것

  1. 타깃 고객이 MOA를 실제로 필요로 하고 반복 사용할 것이다.
  2. MOA가 충동구매를 사전에 막는다.
  3. 캐릭터가 리텐션을 높인다.
  4. AI 분석이 경쟁사보다 정확하다.
  5. 1년 안에 특정 가입자·매출을 달성한다.
  6. 검증된 유입 채널이나 수익모델이 있다.
  7. 복제하기 어려운 불공정 우위가 있다.
캔버스 결론

현재 MOA는 문제–고객–해결책에 대한 구체적인 가설과 이를 시험할 MVP 설계까지는 갖췄습니다. 반면 채널–수익원–불공정 우위는 근거가 없어 미확인이고, 비용 구조와 핵심 지표도 항목만 정의됐을 뿐 실제 값이 없습니다. 그래서 지금 할 일은 사업 모델을 그럴듯하게 채우는 것이 아니라, 다음 두 가지를 먼저 확인하는 것입니다.

확인 과제 01

타깃 고객이 지킬 돈 한 숫자 + 부분 절감 + 비난 없는 성장을 이해하고, 실제로 다음 소비를 조정하는가.

확인 과제 02

한 달 뒤에도 다시 챕터를 시작할 만큼 가치가 남는가.

이 두 가설이 확인되기 전까지 채널 효율, 유료화, 매출, 불공정 우위를 확정 사실처럼 사용하지 않습니다.
