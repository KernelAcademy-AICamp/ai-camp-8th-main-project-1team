> **정합 메모**
> `사용 한도 = 기준 소비 × (1 − 강도)`, `지킬 돈 = 기준 소비 × 강도`,
> `지킨 돈 = max(지킬 돈 − max(누적 소비 − 사용 한도, 0), 0)`로 통일한다.
> 자동 분석·부분 차감 구조는 유지한다.
> 여행·물건 목표, 목적지, 보상, 참기 세션은 제거하고 월말 확정 지킨 돈만 캐릭터 성장에 누적한다.

이 워크시트는 기능 기획을 구현 가능한 스펙으로 점검하는 도구다. `_미작성_`은 아직 결정이 필요한 항목이다.

## Part 1. 데이터 전제 — 소비 데이터는 어떻게 들어오는가

### Q1. 데이터 연동 방식은 무엇인가?

- 실제 마이데이터 사업자를 연동하지 않는다.
- 카드 승인내역과 은행 계좌 거래내역 응답 스키마를 모사한 더미 서버를 구축한다.
- 페르소나별 3개월 이상 거래로 분석·분류·차감·월말 리포트를 재현한다.
- 동의 플래그·페이지네이션·부분 실패·취소·정정을 실제처럼 처리한다.
- 계좌 거래는 고정지출·구독 탐지에만 사용하고 누적 소비에는 반영하지 않는다.

### Q2. 실제 API 연결이 MVP의 전제인가?

- 아니다. 더미 서버가 1차 구현의 기본 플랜이다.
- 표준 필드와 pull 방식 제약은 문서로 검증한다.
- 실연동에서 결제 직후 데이터 수신을 보장할 수 없으므로 MVP 설명에도 `실시간 금융 알림`이라고 쓰지 않는다.

### Q3. 인증과 개인정보의 경계는?

- 가상 CI와 더미 이름·전화번호로 인증 화면만 시뮬레이션한다.
- 더미 이름·전화번호는 CI 반환 즉시 파기한다.
- 실제 CI 생성·검증, 실제 개인정보 수집·저장, 인증서 검증은 하지 않는다.

### Q4. 연동이 실패하면?

- 기관별 성공분을 먼저 반영하고 실패한 기관만 재시도한다.
- `/demo/*`에 시계 이동·거래 주입·초기화 기능을 둔다.
- 발표에서 `한도 이내 → 침묵`과 `한도 초과 → 초과분 차감·알림`을 모두 재현한다.

## Part 2. 자동 판정 — 시스템은 어떻게 지킴 상태를 아는가

### Q5. 선택 카테고리 소비는 어떻게 반영하는가?

```text
신규 카드 거래
→ 소비 분석 Agent가 카테고리 분류
→ 선택 카테고리인지 대조
   ├─ 아니오: 침묵
   └─ 예: 합계 누적 소비에 가산
          ├─ 합계 사용 한도 이내: 화면 금액 유지 · 침묵
          └─ 합계 사용 한도 초과: 초과분만 지킨 돈에서 차감 · 알림
```

- 사용자는 거래마다 확인하지 않는다.
- 오분류 수정과 거래 취소·환불만 금액을 복구한다.
- 맥락 태그는 다음 달 분석용이며 금액을 면제하지 않는다.

### Q6. 거래와 카테고리는 어떻게 매핑하는가?

- 1차: 가맹점명 키워드 사전
- 2차: ML 가맹점명 텍스트 분류
- 3차: LLM 분류 + 불확실 표시 + 사용자 수정
- 사용자 수정 결과를 규칙 사전 또는 학습 데이터에 반영한다.

### Q7. 합계 차감인가, 카테고리별 차감인가?

- 사용자 차감은 **선택 카테고리 합계 기준**이다.
- 카테고리별 한도 상태는 내부 분석에만 보관한다.
- 한 카테고리가 자체 한도를 넘었어도 전체 합계 사용 한도 안이면 지킨 돈은 줄지 않고 알림도 없다.

## Part 3. Agent 정의서

### 3-1. 소비 분석 Agent

- **트리거**: 온보딩 · 신규 거래 · 월말
- **입력**: 카드·계좌 거래 · 전월 지킴 실적 · 맥락 태그 · 오분류 이력
- **출력**:
  - 카테고리별 기준 소비
  - 이번 달 줄여볼 카테고리 후보
  - 권장 강도·추천 근거·신뢰도
  - 반복 결제·요일·시간대 패턴
  - 계좌 기반 고정지출·구독
  - 가치 소비 보호 신호·성향 요약
- **LLM**: 애매한 거래 재분류 · 추천 근거 · 성향 문장
- **규칙**: 집계·평균·빈도 · 키워드 분류 · 계좌 적요 분류
- **하지 않음**: 카테고리 자동 확정 · 강도 확정 · 소비 목표 추천

### 3-2. 지킴·성장 Agent

- **트리거**: 카테고리·강도 확정 · 신규 거래 · 재분류 · 취소·환불 · 월말 · 캐릭터 생성
- **입력**: 분석 결과 · 사용자 선택 · 신규 카드 거래 · 재분류 · 캐릭터 이름
- **출력**:
  - 내부 사용 한도
  - 지킬 돈
  - 누적 소비·남은 한도
  - 현재 지키는 금액
  - 월중 상태·실시간 차감 알림
  - 월말 확정 지킨 돈
  - 캐릭터·레벨·외형·월간 챕터·누적 지킨 돈(성장 상태)
- **LLM**: 비난 없는 실시간 차감·회복 문구
- **규칙**:

```text
사용 한도 = 기준 소비 × (1 − 절약 강도)
지킬 돈   = 기준 소비 × 절약 강도
남은 한도 = max(사용 한도 − 누적 소비, 0)
지킨 돈   = max(지킬 돈 − max(누적 소비 − 사용 한도, 0), 0)
초과 지출 = max(누적 소비 − 기준 소비, 0)
이번 달 성장 반영액 = 월말 확정 지킨 돈 (멱등)
```

- **하지 않음**: LLM 금액·레벨 계산 · 태그에 따른 면제 · 리포트·추천 생성(③ 담당)

### 3-3. 취향·추천 Agent

- **트리거**: 주간 · 월말 · 리포트·추천 화면 진입
- **입력**: 마이데이터(취향 분석용) · ①의 소비 분석 요약(성향 프로필·충동구매지수 포함) · ②의 지킴·성장 상태 · 외부 실제 금융상품 정보
- **출력**:
  - 취향·성향 분석
  - 주간 리포트
  - 월말 리포트 (지킴 성과·성장 서술)
  - 맞춤 금융상품 정보성 비교 추천
- **LLM**: 리포트·성장·회고 서술 · 추천 사유 문장
- **규칙**: 리포트 집계 · 취향 특징 추출 · 상품 매칭·정렬 (방식 미정)

- **하지 않음**: ②의 금액·성장 값 변경 · 목적지·소비형 보상 · 금융상품 판매·중개·가입 대행 · 목표·페이스 재도입

## Part 4. Agent 간 연결

| 연결 | 전달 데이터 | 갱신 주기 |
|---|---|---|
| 분석 → 지킴 (A) | 기준 소비·절약 후보·권장 강도·추천 근거·분류 결과 | 월초 · 거래 증분 |
| 분석 → 취향·추천 (B) | 소비 분석 요약(월평균·반복 결제·패턴·후보+예상액·성향·충동지수) | 월초·주간·월말 |
| 지킴 → 취향·추천 (C) | 지킨 돈·차감·월중 상태·월말 확정·누적 지킨 돈·성장 레벨 | 상태 변경 시·월말 |
| 외부 → 취향·추천 (D) | 실제 금융상품 정보(실 금리 등, 정보성) | 갱신 시 |
| 분석 → 지킴 (F) | 오분류 재분류 결과 | 수정 즉시 |
| 지킴 → 분석 (G) | 월말 실적·맥락 태그·오분류율 | 월말 |

성장은 ② 내부에서 처리한다(별도 ②→③ 성장 반영·③→① 요약 없음). 월말 순서는 `지킴 확정·성장 누적 → 다음 달 분석`이며, ③의 리포트·추천은 사용자 산출물이다. 상세 스키마와 쓰기 권한은 `07_에이전트_연결부.md`를 따른다.

## Part 6. 월간 상태와 성장 전이

```text
카테고리·강도 선택 중
→ 지키는 중
→ 월말 지킨 돈 확정
→ 캐릭터 성장·챕터 기록
→ 다음 달 카테고리·강도 선택
```

내부 분석 상태:

| 결과 | 조건 |
|---|---|
| full | 누적 소비 ≤ 사용 한도 |
| partial | 사용 한도 < 누적 소비 < 기준 소비 |
| missed | 누적 소비 ≥ 기준 소비 |

사용자 리포트는 내부 상태를 실패 낙인으로 전면 노출하지 않고 `이번 달 {금액}원을 지켰어요`를 중심으로 구성한다.

### Q10. 중간에 소비한 사용자에게 무엇을 보여주는가?

- 한도 이내면 Home 금액을 유지하고 알리지 않는다.
- 한도를 넘겼어도 현재 지키는 금액이 남아 있으면 그 금액을 계속 보여준다.
- 0원이 되어도 월간 경험을 조기 종료하지 않는다.
- 불가피 태그가 반복되면 다음 달 다른 카테고리나 낮은 강도를 제안한다.

### Q11. 부분 달성 금액은 어떻게 처리하는가?

- 1차: 월말 확정 지킨 돈으로 리포트와 성장 원장에 반영한다.
- 2차: 0원보다 크면 내부 결과 상태와 관계없이 가상 지킴 계좌로 옮긴다.

### Q12. 성장은 언제 확정하는가?

- 월중 상태는 캐릭터 연출에만 사용한다.
- 월말 원장 마감 뒤 확정 지킨 돈을 한 번 누적한다.
- 같은 월의 멱등키는 재반영하지 않는다.
- 성장·레벨·외형은 감소하거나 리셋되지 않는다.

## Part 7. 소비 목표 비개입과 가상 계좌

### Q13. 사용자의 목표는 무엇인가?

- 서비스가 여행·물건 목표를 제시하지 않는다.
- 제품 안의 목표는 `아낄 수 있는 돈을 아끼는 행위` 자체다.
- 이를 캐릭터가 돈을 지키며 성장하는 여정으로 동기 치환한다.

### Q14. 지킨 돈의 사용처는 어떻게 다루는가?

- 사용자의 자유다.
- 소비형 보상·상점을 추천하지 않는다. 금융상품은 정보성 비교만 제공한다(판매·중개 없음, `가입은 각 사로`).
- 사용처를 캐릭터 성장 조건과 연결하지 않는다.

### Q15. 가상 지킴 계좌는 언제 붙는가?

- 2차 구현에서 별도 가상 금융 서버로 구현한다.
- 월말 지킨 돈 확정 후 멱등키를 포함해 이체한다.
- 누적 잔액만 표시하며 목표 진척률은 만들지 않는다.

## Part 8. 데이터·데모 준비

### Q16. 시연용 거래는 어떻게 만드는가?

- 페르소나별 3개월 이상 소비 패턴을 설계한다.
- 택시·배달 등 줄이기 후보와 건강·가족·취미 등 보호할 가치 소비를 함께 넣는다.
- 가맹점명은 명확·PG사·애매한 이름을 섞는다.
- 계좌 거래에는 급여·카드 대금·고정지출·구독 자동이체를 넣는다.
- 시드를 고정해 추천과 월말 결과를 재현한다.

### Q17. 발표 데모 시나리오는?

```text
마이데이터 연결
→ AI 분석
→ 줄일 카테고리·강도 50% 선택
→ 지킬 돈 115,000원 확정
→ 캐릭터 이름 설정 · 7월 챕터 시작
```
