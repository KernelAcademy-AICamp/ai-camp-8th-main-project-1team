# 09. IA — 정보 구조

> 프로젝트명: MOA
> 기준 문서: `06_prd.md` v9 · `07_에이전트_연결부.md` v5
> 상태: IA v4 · 소비 분석(①) + 지킴·성장(②) + 취향·추천(③) 3-Agent 반영
> 최종 업데이트: 2026-07-24

이 문서는 MOA MVP의 화면 구조, 진입 조건, 핵심 정보, 화면 간 이동을 정의한다.

## 1. 구조 원칙

### 1.1 상시 탐색은 3개 탭으로 제한한다

| 탭 | 사용자가 답을 얻는 질문 |
|---|---|
| **홈** | 지금 얼마를 지키는 중인가? |
| **리포트** | 이번 주·이번 달에는 어떻게 지켰는가? |
| **마이** | 내 캐릭터와 설정은 무엇인가? |

월초 설정과 월말 확정은 매일 찾는 목적지가 아니라 시점에 따라 열리는 인터럽트 플로우다. 별도 탭을 만들지 않는다.

### 1.2 Home의 금액 주 지표는 하나다

- Home에는 `현재 지키는 금액`만 큰 금액으로 표시한다.
- 기준 소비·사용 한도·남은 한도·소진률·목표선을 표시하지 않는다.
- 캐릭터 영역에는 레벨만 요약하고, `누적 지킨 돈`은 성장 상세에서 확인한다.
- 한도 이내 거래는 화면 금액을 바꾸지 않고 알림도 보내지 않는다.

### 1.3 자동 원장과 캐릭터 성장을 분리한다

```text
월중 자동 원장
거래 → 분류 → 합계 한도 비교 → 초과분 차감 → 현재 지키는 금액

월말 성장 원장
최종 지킨 돈 확정 → 누적 지킨 돈 반영 → 레벨·외형 갱신
```

월중 거래가 들어올 때마다 성장시키지 않는다. 월말 확정 전 수정·환불로 값이 바뀔 수 있기 때문이다.

### 1.4 소비 목표는 만들지 않되, 금융상품은 정보성 비교로 제공한다

여행·물건 목표, 목적지, 도착률, 남은 거리, 소비형 보상·상점은 IA에 포함하지 않는다. 지킨 돈의 사용처는 사용자의 자유다.

금융상품은 예외다. 아낀 돈의 보관처(저축)를 다루는 **정보성 비교 추천**은 ③이 담당하며 화면(FP)에 포함한다. 판매·중개·가입 대행은 하지 않고 `가입은 각 사로` 안내하며, 목표 금액·달성 페이스로 제시하지 않는다.

### 1.5 캐릭터는 실패를 연기하지 않는다

- 캐릭터는 지출에 슬퍼하거나 약해지지 않는다.
- 월말 지킨 돈이 0원이어도 레벨과 외형을 유지한다.
- 내부 `full/partial/missed` 상태는 분석용이며 사용자 메시지는 지킨 금액 중심이다.

## 3. 핵심 사용자 흐름

### 3.1 최초 진입

```text
연결 안내
→ 전송요구·가맹점명 동의
→ 기관 선택·가상 인증
→ 거래 조회
→ AI 소비 분석
→ 가치 소비 확인
→ 줄일 카테고리 선택
→ 절약 강도 선택
→ 지킬 돈 확인
→ 캐릭터 이름 설정
→ 첫 월간 챕터 시작
→ Home
```

캐릭터 설정은 절약 구조를 이해한 뒤 한 번만 받는다. 캐릭터를 만들기 위해 여행·물건 목표를 입력하게 하지 않는다.

### 3.2 월중 자동 지킴

```text
신규 거래 수신
→ 자동 분류
→ 선택 카테고리인가?
   ├─ 아니오: 기록만 저장 · 침묵
   └─ 예: 합계 누적 소비 갱신
          ├─ 합계 사용 한도 이내: Home 금액 유지 · 침묵
          └─ 합계 사용 한도 초과: 초과분만 차감 · 인앱 알림
                                      ├─ 맥락 태그 선택
                                      └─ 오분류 수정
```

### 3.3 월말 성장

```text
월말 원장 마감
→ 최종 지킨 돈 확정
→ RP-02 월말 리포트
→ ③이 확정 금액을 성장 원장에 1회 반영
→ RP-03 캐릭터 성장 연출
→ 챕터 기록 저장
→ RP-04 다음 달 지킬 것 정하기
```

### 3.4 월초 재진입

```text
지난달 챕터 요약
→ 이번 달 AI 추천 확인
→ 카테고리·강도 재확정
→ 새 챕터 시작
→ Home
```

누적 레벨·외형은 그대로 유지한다.

## 5. 화면별 정의

## 5.1 온보딩

### ON-01 · 스플래시·시작

| 항목 | 정의 |
|---|---|
| 목적 | MOA의 핵심 가치를 한 문장으로 전달하고 연결 플로우를 시작한다 |
| 핵심 문구 | `아낄 수 있는 돈을, 캐릭터와 함께 지켜봐요.` |
| 액션 | `시작하기` → MD-01 |
| 금지 | 수익률·보상·여행·물건 목표 강조 |

### MD-01 · 마이데이터 안내

| 항목 | 정의 |
|---|---|
| 목적 | 왜 카드와 계좌 데이터를 불러오는지 설명한다 |
| 표시 | 최근 3개월 소비 분석 · 카드 거래 자동 반영 · 계좌는 고정지출 탐지에 사용 |
| 안내 | 포트폴리오용 더미 데이터 시뮬레이션임을 명시 |
| 액션 | `연결 계속하기` → MD-02 |

### MD-02 · 전송요구서·동의

| 항목 | 정의 |
|---|---|
| 목적 | 카드·은행 정보 수집 항목과 기간을 확인받는다 |
| 필수 | 카드 목록·승인내역 · 계좌 목록·거래내역 · 가맹점명 |
| 표시 | 정보 제공자 · 수신자 · 목적 · 보유기간 · 전송주기 |
| 액션 | 전체 필수 동의 → MD-03 |

### MD-02E · 가맹점명 동의 거부 차단

| 항목 | 정의 |
|---|---|
| 조건 | 가맹점명 선택 동의가 꺼진 상태로 계속하기 |
| 문구 | `가맹점 정보 없이는 소비 카테고리를 분석할 수 없어요. 동의가 필요해요.` |
| 액션 | `다시 확인하기` → MD-02 |

### MD-03 · 기관 선택

| 항목 | 정의 |
|---|---|
| 목적 | 연결할 카드사와 은행을 선택한다 |
| 표시 | 업권별 기관 목록 · 전체 선택 · 선택 수 |
| 액션 | `선택한 기관 연결` → ON-02 |

### ON-02 · 가상 본인인증

| 항목 | 정의 |
|---|---|
| 목적 | 표준 연결 경험을 재현한다 |
| 표시 | 더미 이름·전화번호 · 인증 코드 |
| 처리 | 가상 CI 반환 후 이름·전화번호 즉시 파기 |
| 액션 | 인증 완료 → MD-04 |

### MD-04 · 조회 진행

| 항목 | 정의 |
|---|---|
| 목적 | 기관별 카드·계좌 조회 진행 상태를 보여준다 |
| 표시 | 대기·조회 중·완료·실패 · 업권별 진행률 |
| 완료 | 성공 기관이 1개 이상이면 분석 시작 |
| 실패 | 실패 기관이 있으면 MD-04E |

### MD-04E · 일부 기관 실패·재시도

| 항목 | 정의 |
|---|---|
| 목적 | 성공분을 잃지 않고 실패한 기관만 다시 연결한다 |
| 액션 | `실패 기관 다시 시도` · `성공한 데이터로 계속` |

### AN-01 · 소비 분석 요약

| 항목 | 정의 |
|---|---|
| 목적 | AI가 찾은 반복 소비와 조정 가능한 카테고리를 보여준다 |
| 표시 | 카테고리별 월평균 경향 · 반복 패턴 · 추천 후보 · 추천 근거 |
| 표현 | AI 제안을 정답이나 낭비 판정처럼 말하지 않는다 |
| 액션 | `내 기준 확인하기` → AN-02 |

### AN-02 · 가치 소비 확인

| 항목 | 정의 |
|---|---|
| 목적 | 사용자가 줄이고 싶지 않은 소비를 보호한다 |
| 질문 | `이 소비는 나에게 중요한가요?` |
| 처리 | 중요하다고 선택한 신호는 후보 우선순위를 낮춘다 |
| 금지 | 이 데이터를 목표·보상 추천에 사용 |
| 액션 | `계속` → CT-01 |

### CT-01 · 줄일 카테고리 선택

| 항목 | 정의 |
|---|---|
| 목적 | 이번 달 자동 추적할 카테고리를 사용자가 확정한다 |
| 표시 | AI 추천 후보 · 추천 근거 요약 · 신뢰도 낮음 표시 |
| 기본 | AI 추천은 미리 선택할 수 있으나 사용자가 해제 가능 |
| 가이드 | 1~2개 권장 |

### MO-01 · 월초 진입

|---|---|
| 조건 | 새 달 첫 진입 · 전월 월말 리포트 완료 후 |

| 액션 | `이번 달 다시 정하기` → CT-01 |
| 미완료 | 전월 설정을 임시 승계하고 HM-01A 배너 표시 |

### HM-01 · Home

| 영역 | 표시 |
|---|---|
| 캐릭터 여정 | 캐릭터 · 이름 · Lv.n · `{월}의 길을 지키는 중` |
| 주 지표 | `100,000원 지키는 중` |
| 월중 상태 | `지키는 중` 또는 `지킬 돈 소진` |
| 선택 정보 | 선택 카테고리·강도 칩 |
| 최근 변화 | 마지막 차감 또는 복구 1건 |
| 바로가기 | 거래 내역 · 알림함 · 주간 리포트 |

Home에서 숨기는 정보:

- 기준 소비
- 사용 한도
- 남은 한도
- 소진률·목표선
- 누적 지킨 돈
- 목적지·남은 거리·보상

누적 지킨 돈을 보려면 캐릭터 영역을 눌러 CH-02로 이동한다. 현재 지키는 금액과 장기 누적 금액을 같은 위계에 놓지 않기 위해서다.

### HM-01A · 전월 설정 승계 배너

|---|---|

| 액션 | `확인하기` → MO-01 |

### HM-02 · 카테고리 상세

| 항목 | 정의 |
|---|---|
| 표시 | 카테고리·강도 · 관련 거래 · 각 거래의 차감 기여액 |
| 숨김 | 카테고리별 내부 한도 금액 |
| 액션 | 거래 선택 → TX-02 |

### CH-02 · 캐릭터 성장 상세

| 항목 | 정의 |
|---|---|
| 목적 | 장기 여정과 성장 이력을 확인한다 |
| 표시 | 캐릭터 외형 · 이름 · Lv.n · 누적 지킨 돈 · 다음 성장 구간 |
| 표시 | 월별 챕터 카드 · 각 달 최종 지킨 돈 |
| 액션 | 이름 변경 → CH-03 · 챕터 → RP-06 |
| 금지 | 목적지·도착률·구매 가능한 아이템 |

`다음 성장까지 {금액}`은 캐릭터 성장 구간 정보일 뿐 소비 목표가 아니다. 사용자 테스트에서 새 금액 목표처럼 오해되면 구간 숫자를 숨기고 진행 바만 표시한다.

### RP-03 · 월말 성장 연출

|---|---|
| 입력 | 월말 확정 지킨 돈 · 성장 멱등키 |
| 표시 | 이번 달 성장 반영액 · 누적 지킨 돈 · 레벨 변화 · 외형 변화 |

| 레벨 유지 | 챕터 기록 추가 연출 |
| 0원 | `이번 챕터의 성장은 쉬어가요. 지금까지의 여정은 그대로예요.` |
| 액션 | `다음 달 준비하기` → RP-04 |

동일한 월의 성장 이벤트는 다시 진입해도 연출만 재생하고 금액은 중복 반영하지 않는다.

| 취향 분석·금융상품 정보성 추천 | FP-01 · FP-02 |
| 다음 달 루프 | RP-04 · MO-01 |
| 데모 패널 | DM-01 |

삭제된 기능에는 대응 화면을 만들지 않는다.

| 삭제 기능 | 삭제 화면 |
|---|---|
| 여행·물건 목표·목적지 | 목표 설정·현황·변경 화면 |
| 참기 세션 | 세션 설정·진행·종료 화면 |
| 소비형 보상·상점 | 보상·상점 추천 화면 |
| 지킨 날·쿠폰 | 일일 성장·쿠폰 화면 |
| OS 위젯 | 위젯 추가·설정 화면 |

금융상품은 삭제 대상이 아니다. **정보성 비교 추천**은 FP-01/FP-02로 포함하며(판매·중개 화면은 없음), 상세는 §5.4·§8.2를 따른다.

② 지킴·성장은 금액 로직과 캐릭터·성장 **상태(계산)** 를 소유하고, ③ 취향·추천은 그 상태를 **리포트로 서술**하며 취향 분석과 금융상품 정보성 추천을 만든다. RP-03은 성장 결과(②)를 ③이 연출·서술한다.
