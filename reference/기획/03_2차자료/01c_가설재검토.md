# 01c. 가설 재검토

> 목적: 01·01a·01b의 리서치 결과를 바탕으로 문제 가설별 검증 수준과 우선순위를 최종 정리한다.
> 작성일: 2026-07-21
> 상태: 최종 방향 반영본

## 읽기 전에

각 가설은 아래 기준으로 확정 수준을 구분한다.

- **확정**: 현재 팀의 공통 방향으로 채택
- **근거 확보**: 데스크 리서치에서 반복적으로 확인
- **가설**: 제품 검증 전까지 잠정 채택
- **추가 검증 필요**: 사용자 반응이나 실제 데이터 확인 필요

공감도 수치는 정량·정성 자료를 합쳐 본 탐색적 추정치이며, 모집단을 대표하는 확정 수치가 아니다.

---

## 최종 우선순위

1. **P4 감정·보상 심리로 인한 충동 소비**
2. **P1 소비 규모의 뒤늦은 인지**
3. **P2 무엇을 줄여야 할지 판단하기 어려움**
4. **P5 절약 목표의 중간 성과를 체감하기 어려움**
5. **P7 동의하지 않은 절약 지적에 대한 거부감**
6. **P3 가치 소비와 습관 소비 구분의 어려움**
7. **P6 자동 분류 결과에 대한 불신**

### MVP에서 다룰 핵심 문제 세트

```text
스트레스와 보상 심리로 소비한다 (P4)
→ 나중에 카드값을 보고 놀란다 (P1)
→ 무엇을 줄일지 모른다 (P2)
→ 절약 목표를 세워도 중간에 얼마나 지켜지는지 알 수 없다 (P5)
```

---

## 가설별 재검토

### P4 — 감정·보상 심리로 인한 충동 소비

- 공감도 추정 70~85% · 근거 등급 높음 · **핵심 문제로 채택**
- 계획하지 않은 소비 경험과 "이 정도는 해줘도 된다"는 보상 심리가 반복 확인됐다.
- 의지만으로는 반복 루프를 끊지 못한다. 일회성 분석보다 반복 소비 추적이 필요하다.

### P1 — 소비 규모의 뒤늦은 인지

- 공감도 추정 60~75% · 근거 등급 중상 · **유지**
- 간편결제 확산으로 결제 순간의 감각이 약해지고, 월말·결제일에야 규모를 인지한다.
- 월말 회고만으로는 늦다. 신규 거래 갱신 시 현재 지키는 금액의 변화를 보여줄 필요가 있다.

### P2 — 무엇을 줄여야 할지 판단하기 어려움

- 공감도 추정 50~65% · 근거 등급 중간 · **핵심 기능과 직접 연결** · 추가 검증 필요
- "식비가 많다" 수준의 정보만으로는 행동을 선택하기 어렵다.
- AI가 조정 가능한 소비 후보를 먼저 제안하고, 분석을 정보 제공에서 의사결정 지원으로 확장한다.

### P5 — 절약의 중간 성과를 체감하기 어려움

- 공감도 추정 55~70% · 근거 등급 중간 · **표현 수정 후 유지** · 추가 검증 필요
- 기존 "절약이 자산으로 연결되는 감각 부족"을 사용자 언어에 가깝게 재정의했다.

> 아끼겠다고 결심해도 이번 달에 얼마나 지켜지는지 알기 어렵다.
> 절약 목표가 추상적이라 중간에 쉽게 무너진다.

- 절약을 금액으로 구체화하고, 현재 지키는 금액 하나를 지속적으로 보여준다. 내부 한도 안에서는 침묵하고 초과 시점부터 차감을 알린다.

### P7 — 절약 권유를 잔소리로 느낌

- 공감도 추정 45~60% · 근거 등급 중간 · **양면성을 가진 문제로 유지** · 추가 검증 필요
- 문제는 지적 자체가 아니라, 동의하지 않은 기준으로 지적받는 것이다.
- 전체 소비를 평가하지 않고, 사용자가 줄이기로 선택한 소비만 추적한다.

### P3 — 가치 소비와 습관 소비 구분 어려움

- 공감도 추정 40~55% · 근거 등급 중하 · **잠재 문제로 재정의** · 인터뷰 최우선 검증 대상
- 사용자는 "많이 썼다"는 말은 자주 하지만 "소비 기준이 없다"를 직접 언어화하지 않는다.

> 소비 카테고리만으로는 가치 소비와 조정 가능한 소비를 구분할 수 없다. 사용자의 우선순위와 맥락이 함께 필요하다.

- AI는 줄일 소비를 확정하지 않고 후보로 제안하며, 사용자가 보호한 가치 소비를 이후 분석에 반영한다.

### P6 — 금융 앱 분류·분석 결과 불신

- 공감도 추정 30~45% · 근거 등급 낮음 · **우선순위 하향**
- 분류 불신이 주요 이탈 사유라는 강한 근거는 부족하다. 핵심 문제보다는 사용 과정의 불편 요소로 다룬다.
- AI가 정답을 확정하지 않고 후보를 제안하며, 불확실한 항목은 명확히 표시하고 사용자가 수정한다.

---

## 이 재검토가 확정한 것

- MVP는 P4·P1·P2·P5를 핵심 문제 세트로 다룬다.
- P3는 인터뷰에서 최우선으로 검증한다.
- P6은 핵심 문제에서 제외하고 오류 수정 UX로 대응한다.
- 모든 가설은 실제 사용자 인터뷰와 프로토타입으로 재확인해야 하며, AI 가상 인터뷰 결과를 실제 근거로 사용하지 않는다.
