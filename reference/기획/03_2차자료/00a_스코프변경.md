# 00a. RFP 대비 스코프 변경 정리

> 목적: 초기 RFP 로드맵과 현재 확정 방향(MOA / 지킬 돈 모델)의 차이를 기록하고, 왜 이렇게 결정했는지 근거를 남긴다.
> 작성일: 2026-07-21
> 상태: 확정

## 한 줄 요약

초기 RFP는 **금융상품 비교 추천을 필수 1순위**로 둔 절약 리포트 + 상품 추천 서비스였다.

리서치와 멘토링을 거치며 핵심 문제가 좁혀졌고, MVP의 중심축을 **지킬 돈 설정과 신규 거래 자동 차감 경험**으로 재편했다.

금융상품 추천과 이상거래 탐지는 초기 MVP 범위에서 제외했다. 이는 방향 표류가 아니라, 6주 안에 검증할 핵심 가설을 좁힌 의도된 스코프 결정이다.

---

## 무엇이, 왜 바뀌었나

### 1. 핵심 문제가 좁혀졌다

RFP 시점에는 "공급자 중심 데이터의 한계, 맞춤 인사이트 부재"를 넓게 잡았다.

리서치(`01`~`01c`)를 거치며 핵심 문제가 다음으로 수렴했다.

> 무엇을 줄여야 할지 판단하지 못하고(P2), 절약이 이번 달에 얼마나 지켜지는지 월말 이전에 체감하지 못한다(P5).

금융상품 추천은 이 핵심 문제를 직접 해결하지 못한다. 반면 지킬 돈 설정과 신규 거래 자동 차감은 P1·P2·P5를 정면으로 다룬다.

### 2. 핵심 경험을 신설하고 승격했다

멘토링(`04`)을 통해 "과거 분석 → 지킬 돈 설정 → 신규 거래 자동 차감 → 월말 보호 리포트" 흐름을 MVP 중심으로 확정했다.

이 경험이 RFP에는 없던 신설 축이며, 현재 MVP의 1~4순위 기능을 이룬다.

### 3. 금융상품·소비 목표·보상을 제품 범위에서 제외했다

금융상품 비교는 지킬 돈·거래 자동 차감이라는 핵심 문제를 직접 해결하지 않는다. 여행·물건 목표와 보상 추천은 절약 서비스가 새로운 소비 욕구를 만드는 모순도 낳는다.

따라서 금융상품·소비 목표·보상은 현재 MOA의 제품 범위에서 제외하고, 지킨 돈의 사용처는 사용자 자유로 둔다.

> **PRD v6 갱신(2026-07-23)**: 목표(물건) 기능과 금융상품 정보성 추천은 유지한다. 1차 지킬 돈 모델은 카테고리 전체 절감으로 단순화하고, 부분 절감은 Later로 이동했다.
>
> **PRD v7 갱신(2026-07-24)**: 화면 시안 검토 결과 전체 절감이 전부 아니면 전무 구조를 만든다고 판단해 **부분 절감으로 되돌렸다.** 사용자가 카테고리별 절약 강도를 고르며, 강도 100%가 곧 전체 절감이므로 v6 모델은 폐기가 아니라 특수 케이스로 흡수된다. 함께 **계좌 업권**을 수집 범위에 추가하고(고정지출·구독 탐지용, 누적 소비에는 미반영), **지킴이 캐릭터와 육성 층**을 정식 요소로 신설했다. 근거는 `04_핵심설계결정.md` 4차 검토를 따른다.
>
> **PRD v8 갱신(2026-07-24)**: 자동 분석·부분 차감은 유지하고 동기 부여 층만 바꿨다. 여행·물건 목표, 목적지, 보상·금융상품 추천, 참기 세션을 제거했다. 월말 확정 지킨 돈을 캐릭터의 누적 여정·성장에 반영하며, 매월 새 챕터를 시작해도 레벨·외형은 감소하거나 리셋되지 않는다.

### 4. 이상거래 탐지(FDS)·대안신용평가를 보류했다

RFP의 부가 3순위였으나, MVP 핵심 문제와 연결이 약하고 6주 범위를 벗어난다. 초기 스코프에서 제외한다.

### 5. 데이터 확보 방식을 바꿨다

RFP 초기 구상은 사용자 직접 입력·파일 업로드였다. 거래 자동 차감 경험을 검증하려면 신규 거래 스트림이 필요하므로, **더미 마이데이터 파이프라인** 방식으로 전환했다.

---

## RFP 대비 유지된 것

- 규제 경계: 실제 금융 라이선스·개인신용정보·자금이 필요한 영역 제외. 2차에서 가상 계좌·이체만 재현
- 절약 리포트 자동화: 형태를 월말 보호 리포트로 재편했으나 유지
- 6주 MVP 검증이라는 프로젝트 성격과 포트폴리오 목적
- 타깃 명확성(20~30대 직장인)으로 인터뷰 섭외 용이성 확보

## 기능 요구사항 표 — 갱신본

| 순위 | 기능 | 유형 | 핵심 |
|---|---|---|---|
| 1 | 더미 마이데이터 소비 분석 + AI 절약 후보 추천 | 필수 | 과거 3개월 카드·계좌 분석 → 줄여볼 카테고리 + 기준 소비 + 권장 강도 + 추천 근거 제안 (낭비로 확정하지 않음) |
| 2 | 가치 소비 검수 + 절약 강도와 지킬 돈 설정 | 필수 | 사용자가 줄일 카테고리와 강도를 선택하고 지킬 돈 확정. 사용 한도는 내부 계산 |
| 3 | 신규 거래 시뮬레이션 + 한도 초과분 자동 차감 | 필수 | 신규 거래 매칭 → 한도 초과분만큼 지킨 돈 차감 → 거래 취소 시 복구 |
| 4 | 지킴 현황 시각화 + AI 월말 절약 리포트 | 필수 | 현재 지키는 금액 하나·초과 차감 내역·월말 지킨 돈·다음 달 추천 |
| 4-1 | 캐릭터 여정·누적 성장 | 필수 | 비난 없는 화자이자 유일한 감정 이입 대상. 월말 확정 지킨 돈이 누적되고 월 경계에도 레벨·외형 유지 |
| 5 | 주간 요약·월말 성장 리포트 | 필수 | 주간은 판정 없는 흐름, 월말은 최종 지킨 돈 확정·캐릭터 성장·다음 달 챕터 |
| 6 | 가상 지킴 계좌·이체 | 2차 구현 | 월말 지킨 돈을 가상 계좌로 옮기고 누적 잔액 보관 |
| — | 금융상품·여행·물건 목표·보상 추천 | 제외 | 지킨 돈의 사용처는 사용자의 자유이며 서비스가 관여하지 않음 |
| — | 이상거래 탐지(FDS) / 대안신용평가 | 보류 | 현재 스코프에서 제외 |

### RFP 원안과의 대응

| RFP 원안 | 현재 |
|---|---|
| 1순위(필수) 맞춤형 금융상품 추천 | 제품 범위에서 제외 — 현재 핵심 문제와 직접 연결되지 않고 소비 목표를 새로 만들 수 있음 |
| 2순위(필수) AI 절약 리포트 자동 생성 | 유지 — 월말 보호 리포트로 재편 |
| 3순위(부가) 이상거래 탐지 / 대안신용평가 | 보류 — 스코프 제외 |
| (신설 없음) | 지킬 돈 설정 + 거래 자동 차감을 MVP 핵심으로 신설 |
