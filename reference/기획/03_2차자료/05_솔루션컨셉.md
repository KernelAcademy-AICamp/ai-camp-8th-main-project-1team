# 05. 솔루션 컨셉

> 상태: 최종 컨셉 v6.0 · 자동 지킴 + 캐릭터 여정·성장 모델
> 프로젝트명: MOA
> 팀명: 쌍토끼클럽
> 최종 업데이트: 2026-07-24

## 한 줄 정의

**AI가 사용자의 소비 패턴에서 이번 달 줄여볼 카테고리를 제안하고, 사용자가 카테고리와 절약 강도를 골라 만든 지킬 돈을 실제 소비로부터 끝까지 지켜내도록 돕는 소비 관리 서비스**

### 서비스 철학

> 소비를 무조건 통제하는 것이 아니라, 사용자가 이번 달 지켜볼 소비를 스스로 선택하고 그 돈을 끝까지 보호하도록 돕는다.

### 프로젝트 성격

MOA는 포트폴리오용 금융 서비스 프로토타입이다.

- 실제 개인신용정보와 실제 자금을 취급하지 않는다.
- 더미 마이데이터 서버로 소비 분석과 거래 갱신을 재현한다.
- 세 명의 팀원이 소비 분석(①)·지킴·성장(②)·취향·추천(③)을 각각 Agent로 구현한다.
- 1차 구현은 월말 지킨 돈 확정까지, 2차 구현은 가상 지킴 계좌 개설과 가상 이체까지다.

---

## 우리가 해결하려는 문제

사용자는 스트레스와 보상 심리로 충동 소비를 반복한다.

소비 후 후회하지만 무엇을 줄여야 할지 판단하지 못한다.

기존 금융 앱은 이미 발생한 소비를 사후 분석하지만, 그 분석이 다음 소비 행동의 기준으로 이어지지 않는다.

절약 목표는 추상적이라 이번 달에 실제로 얼마나 지키고 있는지 월말 이전에는 알기 어렵다.

---

## 핵심 인사이트 — 사후 분석에서 사전 선택으로

기존 금융 앱은 발생한 거래를 보여준 뒤 사용자가 스스로 행동을 결정하게 한다.

MOA는 과거 데이터로 이번 달 줄여볼 카테고리를 먼저 선택하게 하고, 사용자가 고른 절약 강도만큼을 `지킬 돈`으로 전환한다.

```text
기존 서비스
발생한 거래 확인 → 카테고리 분석 → 과소비 인지 → 사용자가 행동 결정

MOA
과거 소비 분석
→ AI가 줄여볼 카테고리와 근거 제안
→ 사용자가 카테고리 검수·선택
→ 카테고리별 절약 강도 선택
→ 지킬 돈 확정 (사용 한도는 내부 계산)
→ 한도를 넘긴 만큼 지킨 돈이 깎임
→ 월말 지킨 돈 확정
```

---

## 왜 부분 절감인가

카테고리를 통째로 끊게 하면 목표가 전부 아니면 전무가 된다. 택시를 끊기로 한 달에 늦잠으로 한 번 택시를 타면 그 달의 완전 달성은 그 순간 사라진다. 한 번의 실수가 남은 3주를 무의미하게 만드는 구조는 사용자를 이탈시킨다.

절약 강도는 이 문제를 해결한다. 사용자는 카테고리마다 얼마를 지킬지 고른다.

```text
사용 한도 = 기준 소비 × (1 − 절약 강도)   이번 달 써도 되는 돈
지킬 돈   = 기준 소비 × 절약 강도          이번 달 지키기로 한 돈
```

강도 100%를 고르면 사용 한도가 0원이 되어 카테고리 전체 절감과 완전히 같아진다. **전체 절감은 부분 절감의 특수한 경우다.** 둘 중 하나를 고른 것이 아니라 하나가 다른 하나를 품는다.

### 계산과 노출을 분리한다

부분 절감의 위험은 화면에 기준 소비·사용 한도·누적 소비·남은 한도 같은 숫자가 함께 등장하는 것이다. 사용자는 어떤 돈을 쓰고 어떤 돈을 지켜야 하는지 혼동할 수 있다.

MOA는 사용 한도를 내부 차감 기준으로만 사용한다.

| 구분 | 값 |
|---|---|
| 내부 계산 | 기준 소비 · 사용 한도 · 누적 소비 · 초과액 |
| 사용자 노출 | 현재 지키는 금액 |

기준 소비가 100,000원이고 강도 50%를 골랐다면 내부 사용 한도는 50,000원이지만, 사용자 화면에는 `이번 달 50,000원 지키는 중`만 표시한다. 사용 한도 안에서는 화면 금액이 유지되고 알림도 보내지 않는다. 한도를 넘긴 순간부터 초과액만큼 금액이 줄고 차감 알림이 시작된다.

## 솔루션의 핵심 구조

```text
더미 마이데이터 연결 (카드 + 계좌)
→ AI 소비 패턴 분석
→ 줄여볼 카테고리 + 기준 소비 + 추천 근거 제안
→ 사용자 검수·카테고리 선택
→ 카테고리별 절약 강도 선택
→ 지킬 돈 확정 (사용 한도는 내부 계산)
→ 신규 거래 실시간 시뮬레이션
→ 선택 카테고리 거래면 누적 소비 반영
→ 사용 한도 이내면 침묵
→ 한도를 넘긴 순간부터 지킨 돈 차감·알림
→ 지킨 돈 확정
→ 월말 확정 금액만 캐릭터의 누적 성장에 반영
→ 새 달에는 다음 여정 챕터 시작
```

2차 구현에서는 월말 지킨 돈을 가상 지킴 계좌로 옮겨 누적 잔액으로 보관한다. 지킨 돈의 사용처는 사용자가 정하며 MOA는 관여하지 않는다.

## 사용자가 경험하는 흐름

### 1. 마이데이터 연결

사용자는 자신의 금융 데이터를 불러오는 온보딩을 경험한다. 실제 개인신용정보 대신 표준 API를 모사한 더미 마이데이터가 연결된다. 카드 승인내역과 계좌 거래내역을 함께 조회한다.

### 2. AI 소비 분석

AI는 최근 3개월 거래를 분석해 반복 소비, 시간대 집중 소비, 구독, 조정 가능성이 높은 소비, 가치 소비를 구분한다.

### 3. 줄여볼 카테고리 제안

AI는 커피·배달·택시처럼 조정 여지가 큰 소비를 후보로 제안하고 카테고리별 기준 소비와 추천 근거를 보여준다. 해당 소비를 낭비라고 확정하지 않는다.

```text
택시
"최근 3개월 택시 이용의 68%가 평일 밤 10시 이후였어요."

배달
"금요일 밤 주문이 반복되고 있어요."
```

### 4. 사용자 검수와 선택

사용자는 추천을 수락·거절하고 다른 카테고리를 추가할 수 있다. 1차 구현에서는 1~2개 선택을 권장한다.

### 5. 절약 강도와 지킬 돈 확정

사용자는 카테고리마다 얼마나 줄일지 고른다. 고른 강도는 사용자에게 `지킬 돈`으로 번역된다. 사용 한도는 차감 시점을 정하기 위해 내부에서만 계산한다.

```text
택시 · 강도 50% → 지킬 돈 40,000원
배달 · 강도 50% → 지킬 돈 75,000원

이번 달 지킬 돈 115,000원
```

강도는 월초에 확정하며 월중에는 바꾸지 않는다.

### 6. 신규 거래 수신

더미 마이데이터로 신규 거래가 실시간처럼 전달된다. AI가 거래를 분류하고 선택 카테고리에 해당하는지 판단한다.

### 7. 누적 소비 반영과 초과 알림

선택 카테고리 거래는 자동으로 누적 소비에 반영된다. 선택하지 않은 카테고리는 지킬 돈 계산에 영향을 주지 않는다.

사용 한도 이내 거래는 내부 원장에만 반영하고 사용자에게 알리지 않는다. 한도를 넘긴 순간부터 차감된 금액과 현재 지키는 금액을 알린다.

```text
🚕 택시 18,500원
정해둔 범위를 12,000원 넘었어요
지키는 금액 112,000원 → 100,000원

[🎯 계획했던 소비] [😅 줄이려던 소비] [🚨 불가피한 소비]
[분류가 틀렸어요]
```

- 맥락 태그는 금액에 영향을 주지 않는다.
- 오분류 수정과 거래 취소·환불만 금액을 복구한다.
- 불가피한 소비가 반복되면 다음 달에는 다른 카테고리나 낮은 강도를 제안한다.

### 8. 지킴 현황 시각화

Home에는 현재 지키는 금액 하나만 표시한다. 기준 소비·사용 한도·남은 한도·소진률은 사용자에게 노출하지 않는다.

```text
 8일   사용 한도 이내             115,000원 지키는 중 · 알림 없음
20일   한도 15,000원 초과          100,000원 지키는 중 · 차감 알림
월말   한도 25,000원 초과           90,000원 지킴
```

지키는 금액이 깎이기 시작하는 순간 자체가 한도 초과 신호다. 별도 목표선이나 사전 한도 경고는 두지 않는다.

### 9. 캐릭터 여정과 성장

Home은 캐릭터가 사용자의 돈을 함께 지키며 길을 이어가는 **여정 화면**이다. 여행·물건 같은 목적지는 없고, 캐릭터가 유일한 감정 이입 대상이다.

```text
이번 달 = 지킬 돈을 지키는 하나의 챕터
장기     = 월말에 확정된 지킨 돈이 쌓이는 캐릭터 성장
```

월중에는 캐릭터가 현재 챕터를 지키는 모습을 보여준다. 거래로 바뀔 수 있는 월중 금액을 성장에 바로 넣지 않는다. 월말에 확정된 지킨 돈만 누적 성장에 한 번 반영한다.

```text
이번 달 성장 반영액 = 월말 확정 지킨 돈
누적 지킨 돈       = 이전 누적 지킨 돈 + 이번 달 성장 반영액
성장 레벨          = 누적 지킨 돈이 속한 구간
```

월이 바뀌면 새 챕터가 시작되지만 누적 지킨 돈·레벨·외형은 유지된다. 지킨 돈이 0원인 달에도 성장만 멈추며 차감·퇴화·비난은 없다.

캐릭터는 알림과 회고 문장의 화자이기도 하다. 다만 금액 계산과 판정은 규칙 엔진이 담당하며 캐릭터가 사용자의 소비를 심판하지 않는다.

### 10. 주간 리포트와 월말 리포트

리포트는 두 주기로 제공한다. 한 달은 피드백 주기로 길기 때문이다.

```text
주간 = 요약    소비 흐름 · 현재 지키는 금액. 판정 없음
월간 = 확정    최종 지킨 돈 · 캐릭터 성장 · 다음 달 진입
```

주간 리포트는 이번 주 선택 카테고리 소비 흐름과 현재 지키는 금액을 보여준다. **달성 여부를 판정하지 않고 성장도 확정하지 않는다.**

월말에는 최초 지킬 돈, 차감 내역, 최종 지킨 돈, 카테고리별 실제 소비, 맥락 태그 집계, 캐릭터 성장 결과, 다음 달 추천 카테고리와 강도를 제공한다. 기준 소비와 사용 한도는 사용자에게 노출하지 않는다. 내부 분석용 결과 상태는 유지하되 사용자에게 `실패`를 전면에 내세우지 않고 지킨 금액을 중심으로 말한다.

| 상태 | 조건 |
|---|---|
| 완전 달성 | 누적 소비 ≤ 사용 한도 |
| 부분 달성 | 사용 한도 < 누적 소비 < 기준 소비 |
| 미달성 | 누적 소비 ≥ 기준 소비 |

### 11. 2차 구현 — 가상 지킴 계좌

월말 지킨 돈이 0원보다 크면 내부 결과 상태와 관계없이 가상 지킴 계좌로 옮긴다. 가상 계좌는 잔액 보관 경험만 제공하며 목표 진척률이나 사용처를 제시하지 않는다.

## 핵심 기능 — 1차 구현

1. 더미 마이데이터 연결 및 과거 3개월 거래 조회 (카드 + 계좌)
2. 거래 분류·반복 패턴 분석
3. 줄여볼 카테고리·기준 소비·추천 근거 제안
4. 사용자 검수와 카테고리 선택
5. 카테고리별 절약 강도 선택과 지킬 돈 확정
6. 신규 거래 실시간 시뮬레이션
7. 누적 소비 자동 반영·맥락 태그·오분류 수정·취소 복구
8. Home에 캐릭터 여정과 현재 지키는 금액 표시
9. 지킴이 문장 생성
10. 캐릭터 이름 설정과 누적 성장
11. 월간 챕터와 월말 성장 연출
12. 주간 리포트(요약)와 월간 리포트(확정)
13. 내부 완전 달성·부분 달성·미달성 판정

## Agent와 AI가 하는 일

### 소비 분석 Agent

- 거래 분류와 반복 소비 패턴 탐색
- 줄여볼 카테고리·기준 소비·추천 강도 제안
- 추천 근거와 소비 성향 문장 생성

### 지킴·성장 Agent

- 선택 카테고리 거래 매칭
- 누적 소비·초과액·지킨 돈 갱신
- 실시간 거래 알림 문구 생성(지킴이 화자)
- 캐릭터·레벨·외형·월간 챕터·누적 성장 상태 진행
- 월말 확정 지킨 돈의 누적 성장 반영(멱등)
- 2차 구현에서 규칙으로 확정된 지킨 돈의 가상 이체 요청

### 취향·추천 Agent

- 마이데이터로 소비자 취향·성향 분석
- ①의 소비 분석과 ②의 지킴·성장 성과로 주간·월말 리포트 생성
- 실제 금융상품 정보를 소비 패턴·취향에 맞춘 정보성 비교 추천(판매·중개 없음, `가입은 각 사로`)
- 성장·회고 서술과 추천 사유 문장 생성

> 금액 계산과 상태 판정은 규칙 엔진이 담당한다. LLM은 계산 결과를 결정하거나 변경하지 않는다.

## 반드시 지킬 제품 원칙

- **후보와 근거만 제안한다.** AI는 낭비를 확정하지 않는다.
- **사용자가 선택한 소비만 반영한다.** 전체 소비를 평가하지 않는다.
- **가치 소비의 기준은 사용자에게 있다.**
- **절약을 구체적인 금액으로 만든다.** 추상적 다짐이 아니라 지킬 돈으로 표현한다.
- **얼마나 줄일지도 사용자가 정한다.** 강도를 AI가 대신 확정하지 않는다.
- **Home은 한 숫자로 설명한다.** 현재 지키는 금액만 보여준다. 기준 소비·사용 한도·남은 한도·목표선은 노출하지 않는다.
- **한도 안에서는 침묵한다.** 내부 원장만 갱신하고, 한도를 넘어 지킬 돈이 실제로 줄어드는 순간부터 알린다.
- **지킨 척하지 않는다.** 월중 금액은 `지키는 중`, 월말 확정 금액만 `지킨 돈`으로 표현한다.
- **모든 차감은 설명 가능해야 한다.** 깎인 금액과 한도를 넘긴 금액은 언제나 같다.
- **오류를 쉽게 고친다.** 분류가 틀리면 사용자가 바로 수정할 수 있다.
- **감정 이입 대상은 캐릭터 하나다.** 여행·물건 목표나 별도 보상 대상을 두지 않는다.
- **성장은 확정 금액만 먹는다.** 월말 지킨 돈만 누적해 환불·재분류 때문에 되돌릴 일을 만들지 않는다.
- **여정은 끝나거나 뒤로 가지 않는다.** 매월 새 챕터가 열리고 누적 성장·외형은 유지된다.
- **새 소비를 제안하지 않는다.** 지킨 돈으로 살 소비형 보상·상점을 추천하지 않는다. 다만 아낀 돈의 보관처(저축) 금융상품은 정보성 비교로만 제공한다(판매·중개 없음).

## 검증할 핵심 가설

- **S1. 절약 후보 발견**: 조정 가능한 소비 카테고리와 근거를 제안하면 무엇을 줄일지 쉽게 결정한다.
- **S2. 지킬 돈 프레이밍**: 절약 강도를 하나의 지킬 돈 금액으로 보여주면 이번 달 지킬 금액을 구체적으로 인식한다.
- **S3. 사용자 검수**: AI 후보를 사용자가 선택·수정하게 하면 자동 분류 불신을 줄이고 통제감을 높인다.
- **S4. 강도 선택**: 카테고리와 절약 강도를 사용자가 직접 고르면 이번 달 지킴을 자신의 결정으로 받아들인다.
- **S5. 손실 회피 피드백**: 한도를 넘긴 순간 지킨 돈이 깎이면 월말 이전에 소비 상태를 인지하고 행동을 조정한다.
- **S6. 선택 소비만 추적**: 사용자가 선택한 카테고리만 반영하면 잔소리로 느끼는 문제를 줄인다.
- **S7. 더미 마이데이터 검증**: 실제 연동 없이도 분석·차감·리포트 경험을 검증할 수 있다.
- **S8. 실패 이후 지속**: 한도를 넘긴 뒤에도 지킨 돈이 남으면 사용자는 남은 기간에도 앱을 사용한다.
- **S9. 소비 맥락 태깅**: 금액과 무관한 태그가 다음 달 카테고리·강도 추천의 근거가 된다.
- **S10. 여정 리텐션**: 월이 바뀌어도 리셋되지 않는 캐릭터 성장이 월말 이후 재방문 동기가 된다.
- **S11. 소비 목표 없는 동기 치환**: 여행·물건 목표 없이도 캐릭터의 여정과 성장이 절약 행동의 의미를 강화한다.

## 초기 성공 기준

| 지표 | 기준 |
|---|---:|
| 줄일 카테고리 1개 이상 선택 | 필수 |
| AI 추천 카테고리 수용률 | 측정 |
| `지킬 돈`이 이번 달 지켜야 할 금액임을 이해 | 80% 이상 |
| 차감 이유 설명 가능 응답 | 80% 이상 |
| 카테고리·강도 선택 권한이 충분하다고 평가 | 80% 이상 |
| 완전 달성·부분 달성·미달성 한 사이클 실행 | 가능 |

리텐션은 핵심 검증 질문으로 유지하되 더미 데이터 기반 MVP에서의 측정 방법은 별도 설계한다. 월말 성장 확인률과 다음 달 챕터 시작률이 대리 지표가 될 수 있는지 함께 검토한다.
