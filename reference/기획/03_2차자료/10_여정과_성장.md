# 10. MOA 코어 루프 상세 — 여정과 성장

> 상태: 통합본 v1 · PRD v8 보조 명세
> 최종 업데이트: 2026-07-24
> 정본 우선순위: 계산·범위는 `06_prd.md`, 연결부는 `07_에이전트_연결부.md`, 화면은 `09_IA.md`

## 0. 통합 결정

2026-07-24 수신한 `절약 지킴이 코어 루프 — 여정과 성장` 제안에서 다음 원칙을 채택한다.

- 여행·물건 같은 소비 목표를 제안하거나 설정하게 하지 않는다.
- 아낄 수 있는 돈을 아끼는 행위 자체를 캐릭터의 여정과 성장으로 동기 치환한다.
- 감정 이입 대상은 캐릭터 하나다.
- 실패·지출을 비난하지 않는다.
- 누적 성장·레벨·외형은 감소하거나 리셋되지 않는다.
- 지킨 돈의 사용처는 사용자의 자유이며 서비스가 관여하지 않는다.
- 성장 아이템·상점·결제로 성장을 구매하지 않는다.

다음 제안은 기존 MOA 코어와 충돌하므로 채택하지 않는다.

- 금융 연동·자동 감지를 빼고 사용자 선언과 자기 보고만 사용하는 구조
- 참기 세션, 사후 금액 입력, `지켰어요/썼어요` 확인
- 세션 성공 금액의 즉시 성장 반영
- 세션 타이머용 위젯

최종 코어 루프는 아래와 같다.

```text
더미 마이데이터 자동 분석
→ 사용자가 카테고리·강도 선택
→ 지킬 돈 확정
→ 거래 자동 분류·누적
→ 합계 사용 한도를 넘긴 금액만 지킬 돈에서 차감
→ 월말 최종 지킨 돈 확정
→ 캐릭터 누적 성장
→ 다음 달 새 챕터
```

---

## 1. 경험 구조

MOA의 진행은 월간과 누적의 두 시간축으로 구성한다.

| 층 | 사용자 질문 | 상태 | 월 전환 |
|---|---|---|---|
| 월간 지킴 | 이번 달 얼마를 지키는 중인가? | 지킬 돈이 유지되거나 초과분만큼 감소 | 새로 시작 |
| 누적 성장 | 캐릭터와 얼마나 오래 지켜왔는가? | 월말 확정 지킨 돈이 누적 | 유지 |

외부 목적지와 종착점은 없다. 매월 한 구간을 지나 다음 구간으로 이어지는 열린 여정이다.

---

## 2. 캐릭터

### 2.1 역할

캐릭터는 세 가지 역할만 갖는다.

1. 사용자가 지키기로 한 돈을 함께 지키는 동행자
2. 차감·회고 문장을 비난 없이 말하는 화자
3. 월말 확정 지킨 돈이 쌓일수록 성장하는 유일한 감정 이입 대상

캐릭터는 금액을 계산하거나 사용자의 소비를 판정하지 않는다.

### 2.2 생성

| 항목 | 내용 |
|---|---|
| 시점 | 최초 카테고리·강도·지킬 돈 확정 후 |
| 입력 | 캐릭터 이름 |
| 글자 수 | 최대 10자 |
| 필수 여부 | 필수 |
| 기본값 | 미정 |
| 완료 | 첫 월간 챕터 시작 연출 후 Home |

여행·물건 목표, 목적지, 목표 금액, 보상 선호는 입력받지 않는다.

### 2.3 표현 규칙

- 소비가 발생해도 슬프거나 화내지 않는다.
- 지킨 돈이 줄어도 캐릭터 외형은 변하지 않는다.
- 월말 지킨 돈이 0원이어도 약해지거나 떠나지 않는다.
- 레벨 상승은 월말에만 연출한다.
- 성장 단계는 결제나 아이템으로 구매할 수 없다.

## 3. 월중 여정

### 3.1 월초

```text
AI가 줄여볼 카테고리와 근거 제안
→ 사용자 카테고리 선택
→ 카테고리별 절약 강도 선택
→ 지킬 돈 확정
→ `{월}의 길을 시작해요`
```

새 달의 챕터가 시작돼도 캐릭터 레벨과 외형은 유지한다.

### 3.2 월중

월중 여정은 새로운 금액을 적립하는 과정이 아니라 월초에 정한 돈을 지키는 과정이다.

```text
지킨 돈 = max(지킬 돈 − max(누적 소비 − 사용 한도, 0), 0)
```

- 합계 사용 한도 이내: 지킨 돈 유지, 알림 없음
- 합계 사용 한도 초과: 초과분만 차감, 인앱 알림
- 오분류 수정·취소·환불: 원장 재계산
- 미선택 카테고리 거래: 계산 제외, 알림 없음

월중 캐릭터는 `이번 달의 길을 지키는 중`이라는 상태만 표현한다. 걷는 속도·거리·도착률은 계산하지 않는다.

### 3.3 Home 노출

| 우선순위 | 정보 |
|---|---|
| 1 | 현재 지키는 금액 |
| 2 | 캐릭터·이름·레벨·이번 달 챕터 |
| 3 | 선택 카테고리·강도 |
| 4 | 최근 차감·복구 |

Home에서 기준 소비·사용 한도·남은 한도·누적 지킨 돈·목적지·보상을 숨긴다. 누적 지킨 돈은 캐릭터 성장 상세에서 확인한다.

## 4. 성장 규칙

### 4.1 성장 시점

성장은 월말 확정 후 한 번만 발생한다.

월중 금액은 취소·환불·재분류로 바뀔 수 있다. 이를 즉시 성장에 넣으면 나중에 되돌려야 하므로 누적형 원칙과 충돌한다.

```text
이번 달 성장 반영액 = 월말 확정 지킨 돈
누적 지킨 돈       = 이전 누적 지킨 돈 + 이번 달 성장 반영액
성장 레벨          = 누적 지킨 돈이 속한 구간
```

### 4.2 불변 조건

- `monthly_growth_amount ≥ 0`
- `cumulative_kept_amount`는 감소하지 않는다.
- `level`은 감소하지 않는다.
- 같은 사용자의 같은 월은 한 번만 반영한다.
- 월 전환 시 레벨·외형·누적 지킨 돈을 리셋하지 않는다.
- 내부 `missed` 결과도 이미 쌓인 성장을 깎지 않는다.

### 4.3 멱등성

월말 성장 반영은 다음 멱등키를 사용한다(② 내부 처리).

```text
idempotency_key = {user_id}:{period}:growth
```

동일 키 재요청은 기존 결과를 반환하고 금액을 다시 더하지 않는다.

### 4.4 성장 단계

구체 구간은 미정이며 다음 원칙을 따른다.

- 누적 지킨 돈만 성장 기준으로 사용한다.
- 초반에는 변화를 빨리 경험하고 이후 구간은 완만하게 늘리는 방식을 검토한다.
- 단계 수는 캐릭터 제작 비용과 반복 사용 기간을 함께 고려한다.
- 다음 단계까지 남은 금액이 새 절약 목표처럼 느껴지면 숫자를 숨기고 추상 진행 바로 바꾼다.

예시 구간은 검증용이며 확정값이 아니다.

| 레벨 | 누적 지킨 돈 |
|---|---:|
| Lv.1 | 0원 이상 |
| Lv.2 | 100,000원 이상 |
| Lv.3 | 300,000원 이상 |
| Lv.4 | 700,000원 이상 |
| Lv.5 | 1,500,000원 이상 |

## 5. 월말 챕터

### 5.1 순서

```text
② 월말 원장 마감
→ 최종 지킨 돈 확정
→ 사용자 월말 리포트
→ ③ 성장 원장 반영
→ 레벨·외형 변화 판정
→ 성장 연출
→ 챕터 기록 저장
→ 다음 달 설정
```

### 5.2 사용자 메시지

| 상황 | 문구 예시 |
|---|---|
| 최종 지킨 돈 있음 | `이번 달 90,000원을 지켰어요.` |
| 레벨 상승 | `누적 420,000원과 함께 모아가 Lv.3으로 성장했어요.` |
| 레벨 유지 | `7월의 길이 기록됐어요. 지금까지의 성장은 그대로 이어져요.` |
| 최종 0원 | `이번 챕터의 성장은 쉬어가요. 지금까지의 여정은 그대로예요.` |

`실패`, `목적지 미도달`, `퇴화`, `잃어버림`은 쓰지 않는다.

### 5.3 챕터 기록

| 필드 | 내용 |
|---|---|
| period | `2026-07` |
| protect_amount | 월초 지킬 돈 |
| finalized_kept_amount | 월말 최종 지킨 돈 |
| growth_applied_amount | 성장에 반영한 금액 |
| level_before / after | 성장 전후 레벨 |
| appearance_changed | 외형 변화 여부 |
| selected_categories | 당시 선택 카테고리·강도 |
| recap_message | 월말 회고 문장 |

내부 사용 한도는 기록하되 사용자 챕터 상세에는 표시하지 않는다.

---

## 6. 주간 리포트

주간 리포트는 월말까지 기다리는 시간을 보완하는 요약이다.

| 포함 | 제외 |
|---|---|
| 현재 지키는 금액 | 성장 확정 |
| 이번 주 선택 카테고리 소비 흐름 | full/partial/missed 판정 |
| 차감·복구 내역 | 성공·실패 문구 |
| 캐릭터 여정 문장 | 목적지·예상 페이스 |

사용자가 참은 금액을 사후 입력하거나 수정하는 기능은 두지 않는다.

---

## 7. 2차 가상 지킴 계좌

월말 확정 지킨 돈이 0원보다 크면 별도 가상 금융 서버로 이체한다.

- 실제 금융기관·실제 자금을 사용하지 않는다.
- 성장 반영과 이체는 각각 멱등키를 갖는다.
- 계좌에는 누적 잔액과 월별 이체 내역만 표시한다.
- 여행·물건 목표 진척률은 붙이지 않는다. 금융상품은 ③의 정보성 비교 추천으로만 연계한다(판매·중개 없음, `가입은 각 사로`).
- 계좌 잔액의 실제 사용처는 서비스가 정하지 않는다.

---

## 8. 비기능·정합 요구

- 월중 거래 반영과 월말 성장 반영은 서로 다른 원장을 사용한다.
- 성장 반영은 트랜잭션 또는 동등한 원자성으로 처리한다.
- 성장 이벤트 재진입 시 연출은 재생할 수 있지만 원장은 중복 반영하지 않는다.
- 캐릭터 이름 변경은 레벨·외형·챕터에 영향을 주지 않는다.
- 삭제된 목표·보상·세션 데이터 필드는 신규 스키마에 남기지 않는다.
- 사용자 노출 문구와 분석용 상태값을 분리한다.

## 10. 검증할 질문

- 외부 소비 목표가 없어도 캐릭터가 절약 동기로 작동하는가?
- 사용자는 월중 `지키는 금액`과 장기 `누적 지킨 돈`을 구분하는가?
- 월말에만 성장하는 구조를 자연스럽게 받아들이는가?
- 종착점 없는 월간 챕터가 다음 달 재방문 동기가 되는가?
- 레벨과 외형 변화가 과도한 게임처럼 느껴지지 않는가?
- 0원을 지킨 달의 메시지가 비난 없이 재시작을 돕는가?
- 다음 성장 구간 금액이 새로운 강제 목표로 오해되지 않는가?
