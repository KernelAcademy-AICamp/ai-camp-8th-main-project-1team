## 이 문서를 읽는 법

"무엇을 만들었는가"가 아니라 **"왜 방향을 두 번 바꿨는가"**를 남긴다. 기획 산출물은 보통 마지막 정답만 깔끔하게 남겨서, 결론까지의 판단 — 무엇에 막혔고 무엇을 포기했는지 — 이 보이지 않는다. 이 프로젝트에서 사고가 가장 많이 일어난 곳은 그 **전환의 순간들**이라, 별도로 복원했다.

두 전환을 **같은 틀**로 정리한다: 기존 가설 → 관측된 문제 → 근거 → 결정 → 포기한 것 → 새로 검증할 것.

**근거의 성격 구분** (같은 급으로 읽지 말 것)

- **정량** — 규모를 보조. 출처·모수를 함께 표기.
- **정성(넷노그래피)** — 사용자의 언어와 맥락.
- **멘토 피드백** — 솔루션 구조의 약점 검토. 2026.07.21 멘토링(27분) 실제 발화.
- **향후 인터뷰** — 실제 타깃 적합성 검증. *아직 수행하지 않음.*

> 통계 %는 리터러시 리뷰 + 넷노그래피(63건)를 합친 데스크 추정치로 통계적 대표성이 없다. AI 가상 인터뷰 결과는 근거로 쓰지 않는다.

## 의사결정 구조

팀 3인(이가원·이상현·지윤정)이 리서치·설계를 공동 수행했고, 멘토링(은종쌤, 2026.07.21)이 **전환 2의 계기**였다. 결정 흐름은 *데스크 리서치로 문제를 좁히고 → 팀이 방향안을 만들고 → 멘토링에서 구현·규제·감각의 벽을 검증하고 → 팀이 재설계·스코프를 확정*하는 구조다.

## 한눈에 보는 전체 여정

| | **세대 0 · 완전 초기** | **세대 1 · 1차** | **세대 2 · 2차 (최종)** |
|---|---|---|---|
| 한 줄 정의 | 참았음을 **감지**해 목표 진척으로 | 참았음을 **직접 기록**해 목표 진척으로 | **지킬 돈**을 정하고 실제 소비를 **차감** |
| 신호를 잡는 법 | 데이터가 센서 (감지+확인) | 사람이 센서 (직접 선언) | 실제 거래가 센서 (차감) |
| 프레이밍 | 획득 (오르는 게이지) | 획득 (오르는 게이지) | 보호 (줄어드는 지킬 돈) |
| 데이터 스탠스 | 마이데이터 감지 = 규제 부담 → 회피 | 사용자 직접 입력 | 더미 거래 파이프라인으로 대체 |

- **전환 1 (0→1)**: 같은 '참기' 철학 안에서 **신호 방식만** 후퇴 (팀 자체 결정) — *구현 방식의 변경*.
- **전환 2 (1→2)**: 참기 철학 **자체를** 폐기 (멘토링이 계기) — *제품 메커니즘의 변경*. 이것이 진짜 피봇이다.

## 세대 0 · 완전 초기 — 참았음을 감지해 획득으로

카드에는 **쓴 돈만** 남고 참은 소비는 기록되지 않는다. 이 "기록되지 않는 좋은 선택"을 발견해 **획득**으로 보여주는 것이 기회 가설이었다("배달비 22,000원을 쓰지 않았어요" → "도쿄 여행까지 22,000원 가까워졌어요"). 신호는 성숙도 곡선(수동 선언 → 비지출 감지+확인 → 자동)으로 설계하고, MVP는 Wizard-of-Oz로 경험만 검증하려 했다.

---

## 전환 1 · 감지+확인 → 직접 기록 (0 → 1)

**기존 가설** — 마이데이터로 비지출을 감지해 "참았음"을 만들고 획득으로 보여준다.

**관측된 문제** — 감지가 규제·구현 양쪽에서 막힌다.

**근거**
- *규제*: 은행·카드 자동 수집(마이데이터)은 라이선스 필요, 직접 입력·업로드는 불요. RFP 규칙에도 "송금 기능 배제" 명시(멘토링 13:53 이상현 확인).
- *구현*: 비지출 감지는 가장 무거운 조각(사후성 + '사건의 부재' 감지 + baseline + 귀인 문제). 6주 안에 비현실적.

**결정** — 제품 비전(감지+확인)은 유지하되, MVP 신호 획득을 **사용자 직접 기록**으로 후퇴. *멘토링 이전, 팀 자체 판단.*

**포기한 것** — 비지출 자동 감지의 MVP 구현, 마찰 0 경험.

**새로 검증할 것** — "사용자는 참은 순간을 성취로 **남기고 싶어 하는가**." (자동 처리부터 하면 이 욕구를 확인할 수 없다.)

---

## 세대 1 · 1차 — 직접 기록 + 목표 진척 + 금융상품 추천

핵심 기능은 ① AI 절약 리포트 ② 참았음 직접 기록 + 목표 진척 ③ 목표 기반 금융상품 Top3. 신성불가침 원칙은 "참은 순간에만 진척이 오른다 · 한 번 오른 진척은 절대 안 깎는다 · 목표엔 이름이 있다(도쿄여행·맥북)"였다.

---

## 전환 2 · 참기·직접기록 → 지킬 돈·차감 (1 → 2) ★ 진짜 피봇

**기존 가설** — 참았음을 직접 기록하면 목표 진척으로 성취감이 생겨 절약이 지속된다.

**관측된 문제** — 멘토링에서 두 급소가 드러났다.
- *기록의 불가능성* — 은종쌤(07:55): **"참는 순간을 어떻게 기록해요?"** → 이가원: "그게 저희 제일 큰 챌린지." 팀도 이미 약점으로 인지.
- *가짜 돈, 감각의 괴리* — 은종쌤(12:28): **"내가 카드를 안 긁어서 있는 거니까 사실 난 돈은 없거든… 피부에 안 와닿을 것 같은 느낌."** → 게이지는 실제 절약액과 무관하게 부풀 수 있고, 진짜 돈이 아니라 감각이 약하다.
- 결국 잘한 순간만 모아, 사용자가 애초에 겪는 뒤늦은 인지(P1)·무엇을 줄일지 모름(P2)·중간 성과 부재(P5)를 풀지 못한다.

**근거**
- *정량*: 계획 없는 소비 경험 72.1% · 자제 의향 84.6%(성인 1,000명, 엠브레인/뉴시스) — 의지는 있으나 기준·중간 피드백이 없음을 보조.
- *정성(넷노그래피)*: "다 필요한 소비였어"(뭘 줄일지 모름) · "나름 아끼는데 왜 돈이 안 모일까"(중간 성과 부재) · "거지방 4일째 다 먹었어요"·"무지출 챌린지는 오히려 보상심리 생겨"(외부 통제→리바운드).
- *멘토*: 위 두 급소 → 대안으로 실제 소비 기준의 '지킬 돈'을 제시. 은종쌤(17:57): **"이번 달 이 30만 원은 보호합시다."** (19:49): **"당신이 지킬 돈… 그러면 구현이 쉬워."** 스트레스 저감 근거도 함께(21:15 "기본 생활 말고 딱 아낄 수 있는 요것만 지켜서").

**결정** — 참기·기록을 폐기하고, **AI가 제안한 '이번 달 지킬 돈'을 실제 거래로 차감**하는 모델로 재설계. 멘토의 씨앗을 팀이 이전 리서치·제약에 맞춰 제품 구조로 확장했다.

| 멘토가 제시한 것 | 팀이 재설계한 것 |
|---|---|
| 실제 소비 기준의 '지킬 돈' 프레임 | 과거 3개월 소비 분석 → AI 줄일 소비 후보 제안 |
| 전체 소비가 아닌 '줄일 소비'만 대상 | 가치 소비 보호 · 절약 강도(강/중/약) 선택 |
| 쓰면 깎이는 차감 구조 | 보호율 중심 UI · 환불 시 금액 복구 |
| | 마이데이터→더미 거래로 대체 · 금융상품 추천 Later 전환 |

> 팀이 대안을 그대로 받아 적은 게 아니라 **파급효과를 즉시 해석**했다. 이상현(20:08): "그럼 지출을 다 넣을 필요도 없겠네요." 이가원(22:30): "목표가 명확해지네요 — 목표를 세운다 어쩐다 안 해도 되고." (분석 범위 축소 · 전체 소비 평가 회피 · 추상적 목표 제거를 스스로 도출.)

**포기한 것**
- 참은 행동을 성취로 쌓는 경험
- 항상 오르기만 하는 긍정 게이지
- 도쿄여행·맥북 같은 감정적 목표
- 충동 직전 개입
- 금융상품 추천의 MVP 포함

**새로 검증할 것** (사용성이 아니라 제품 가설을 3층으로)
- *이해*: 사용자가 별도 설명 없이 '지킬 돈'을 자기 말로 설명하는가
- *수용*: AI 제안 중 최소 1개 이상을 실제 줄일 소비로 선택하는가
- *행동 의도*: 다음 달에도 지킬 돈을 설정하겠다고 응답하는가 · 차감 이후 소비 판단을 다시 해보겠다고 느끼는가

### 원칙 반전을 어떻게 정당화했나

세대 1은 "실패해도 쌓인 걸 빼앗지 않는다(손실 프레임 금지)"를 신성불가침으로 삼았는데, 세대 2는 쓰면 지킬 돈이 줄어든다. 이 모순을 셋으로 풀었다. ① 깎이는 대상은 **사용자가 직접 "줄이겠다"고 고른 소비만** — 가치 소비·고정비는 안 건드림(멘토의 스트레스 저감 논리와 일치). ② 화면 중심을 "얼마 깎였나"가 아니라 **"얼마가 아직 보호되고 있나(보호율)"**로. ③ 취소·환불 시 보호 금액 **복구** → 처벌이 아니라 상태 반영.

### 규제 스탠스의 반전

세대 0은 "마이데이터=라이선스"라 회피했는데, 멘토링에서 매듭이 풀렸다. 은종쌤(20:39): "마이데이터를 안 하고 신용카드 데이터 기반으로 하기로 했다고 하면 된다 — 구현이 깔끔." → 실제 사업자 연동이 아니라 **더미 거래 파이프라인**으로 대체한다.

> **정확한 검증 범위**: 실제 금융데이터를 수집하지 않고도, 더미 거래 이벤트로 "분석 → 차감 → 리포트"의 **사용자 경험과 로직**을 검증한다. (데이터 사업·운영 구조 전체를 검증하는 것은 아니다.)

### 정직한 트레이드오프 — 이 피봇의 약한 고리

세대 1은 데스크 리서치에서 **가장 강하게·반복적으로 확인된 P4(감정·보상 충동소비)**를 소비 직전에 잡으려 했다. 세대 2는 사실상 똑똑한 월 예산 + 실시간 소진이라, **충동의 순간엔 개입하지 않고** 상대적으로 빈도는 낮지만 **제품으로 관측·측정 가능한** P1·P2·P5를 다룬다. 즉 가장 잘 검증된 문제에서 물러나, 설계·측정 가능한 인접 문제로 이동한 것 — 방향 표류가 아니라 **의도된 스코프 결정**이다.

> P4 직접 개입은 (a) 참는 시점이 주관적이라 신호를 못 잡고, (b) 얼마 모였는지 측정 불가라 6주 안에 검증 불가능하다. 그래서 P4는 지킬 돈이라는 그릇으로 **간접적으로** 다루고, MVP는 검증 가능한 P1·P2·P5를 정면으로 친다.

## 세대 2 · 2차 MOA — 지킬 돈 + 실시간 차감 (최종)

> AI가 소비 패턴을 분석해 **이번 달 지킬 수 있는 돈**을 제안하고, 거래에 따라 그 돈이 얼마나 보호되고 있는지 보여주는 소비 관리 서비스.

```text
더미 마이데이터 연결 → AI 소비 분석(과거 3개월)
→ 줄일 소비 후보 + 예상 절약 금액 제안 → 사용자 검수(가치 소비 보호 / 줄일 소비 / 절약 강도)
→ 이번 달 지킬 돈 확정 → 더미 거래 이벤트 시뮬레이션 → 줄이기로 한 소비 발생 시 차감
→ 남은 보호 금액·보호율 갱신 → 월말 보호 리포트
```

- **제품 비전**: 실제 마이데이터 거래 연동
- **MVP 구현**: 더미 거래 이벤트를 실시간처럼 시뮬레이션

금융상품 추천은 세대 1의 핵심이었지만 세대 2에서 **Later**로 내렸다. 6주 MVP의 핵심 가설은 "지킬 돈 + 차감"이기 때문이다.

> MOA 상세 요구사항·화면·성공 기준은 `03_2차-기획문서.md`.

## 세 세대를 관통한 불변 원칙

방향은 두 번 바뀌었다. 정확히 말하면 **행동 변화 메커니즘은 바뀌었지만, 사용자 판단권을 존중하는 설계 원칙은 유지됐다.**

1. **AI는 정답이 아니라 후보와 근거를 제안한다.** (멘토 21:46 "프로포즈는 해줘야 돼요… 그래야 인텔리전트")
2. **가치 소비의 기준은 사용자에게 있다.** (멘토 24:22 성향 기반 구분)
3. **사용자가 직접 선택한 것만 반영한다.** 전체 소비를 평가하지 않는다.
4. **지적이 아니라 회고·보호의 언어로 말한다.**

## 한 줄로 남는 두 문장

> 성취감은 만들어도, 사용자가 애초에 겪는 뒤늦은 인지와 무엇을 줄일지 모름을 풀지 못했다. *(솔루션-문제 불일치를 발견하는 능력)*
> 가장 잘 검증된 문제에서 물러나, 설계·측정 가능한 인접 문제로 이동했다. *(제약하에서 스코프를 선택하는 판단력)*
