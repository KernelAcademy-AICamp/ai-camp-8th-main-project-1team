## 1. 공유된 솔루션 아이디어 3종

- **적금 이자 시각화(게임형)**
  - 적금 이자를 하루 단위(약 33원)로 환산해 목표까지 남은 시간이 줄어드는 방식으로 시각화
  - 돈이 점점 쌓이는 연출로 성취감 부여, 안 보이는 것보다 효과 클 것이라는 가정
- **재미있는 잔소리(테크를 통제로 느끼지 않게)**
  - 레퍼런스: 현대카드 '소비 잔소리' 기능(일정액 이상 지출 시 FC 잔소리) + 게임 캐릭터가 잔소리하는 형태
  - 사용자가 직접 금액·문구·카테고리 커스터마이즈 → 현대카드(고정값) 대비 개인화로 거부감↓·재미↑
- **개인화 목표 프레이밍**
  - 페인포인트 7종 입력 → 핵심 문제(뒤늦은 인지·판단 막막) 도출
  - 시간지평 단축: "1년 4,500원" 대신 "이번 달 무심코 결제할 커피값 13.5만원"으로 즉각 인지
  - 저축 대신 실제 구독·통신비와 1:1 대체 인지 유도
  - 목표를 작은 단위로(에어팟·운동화·티켓), 슬로건 "아껴서 부자 되세요"가 아니라 "무심코 새어나간 돈 모아 소소한 사치"
  - 습관 1~2개만 선택해 제한/알림, 휴식일("오늘은 커피 사도 되는 날"), 잘하게 되면 자산형성(배당ETF·적금) 유도

## 2. 동기(목적) 3가지 정리 및 방향 채택

- 조사된 방법 12개 정리 → **9개는 수단**(돈을 안 쓰게 만드는 도구), **3개는 목적**으로 분리
  - **① 목표 지향** — 지킬 돈을 목표에 가까워지게
  - **② 손실 회피** — 지킬 돈을 잃지 않으려는 동기
  - **③ 게임/놀이** — 지키는 행위를 놀이로
- 카피 3종 비교 결과, **손실 회피는 부정 감정 유발**('매운맛' 느낌) → 초기부터 피하려던 방향과 충돌
- **팀 3인 모두 ①목표 지향으로 수렴** (②는 제외, ③게임은 수단으로 활용)

## 3. 코어 슬라이스(핵심 유저 플로우) 확정 — 멘토 정리

- **온보딩 구간**
  - 마이데이터 자동 연동(사용자 업로드 없이 동의 후 수집)
  - AI 소비 분석 리포트 → "당신은 이런 소비 성향, 아낄 수 있는 항목·금액" 제시 (첫인상이므로 리포트 포맷이 핵심)
  - 사용자가 아낄 항목 체크 → 금액이 실시간으로 올라가는 연출 → "한 달에 이만큼 아낄 수 있습니다"
  - 지킬 돈 확정(예: 30만원) + 목표·보상 동시 세팅(30만원대 물건 몇 개: 에어팟·워치·여행 / 1·2·3개월 경과 옵션)
- **운영 구간**
  - 홈 화면에 목표(지킬 돈) 금액 상시 노출
  - 핵심 액션 2가지 = 돈이 **사라지는 순간 / 지켜지는 순간** → 이 둘을 어떻게 인식시킬지가 코어 기능
  - 이후 트랜잭션(소비·차감)은 Supabase에 기록
- 멘토 조언: 한꺼번에 다 설계하면 구현 못 하고 말만 하게 됨 → **코어만 먼저**, 보상 다양화·주 단위·성향별 톤은 확장 단계로

## 4. '참았음' 신호 처리 방식 재확정

- 참은 순간 감지·등록은 **불가능 영역으로 판단 → 등록 플로우 없음**
- 대신 마이데이터 연동 기반:
  - 안 쓰면 지켜진 것으로 처리
  - 쓰면 알림 → "아끼기로 한 항목이 맞는지" 마지막 1회 확인 (예: 미팅용 커피면 X=지켜짐 / 아끼려던 커피면 차감)
  - 차감 예시: 3,000원 차감 → 30만 → 29만 7천원, 항목 리스트 지속 노출

- **알파 버전 v0.6** 목표 — 가장 기본 MVP만, **다음 주 중반** 산출 목표

## 6. 업무 분배 (3개 에이전트 병렬 개발)

세 에이전트 모두 **마이데이터 분석**을 공통 입력으로 하되, 각자 하는 일이 갈림.

- **① 소비 분석 agent — 이상현**
  - 마이데이터를 분석하여 소비 패턴을 찾고, 불필요한 소비 카테고리를 제안
- **② 목표 금액 지킴이 agent — 지윤정**
  - 마이데이터를 분석하여 사용자의 성향을 파악해, 어떻게 목표 금액을 지키게 할지 정하게 하기
- **③ 보상체계 및 관리 agent — 이가원**
  - 마이데이터를 분석하여 소비 패턴 중 좋아하는 항목을 파악하여, 목표 금액에 맞게 목표 설정을 제안
- 소비 분석(①) 결과가 파이프라인으로 ②·③에 전달됨. 각자 상세 스펙 작성 후 재논의

## ✅ 결정사항

- 동기 방향 **'목표 지향'으로 확정** (손실 회피 제외, 게임은 수단으로 활용)
- **코어 슬라이스 픽스**: 마이데이터 연동 → AI 분석·리포트 → 불필요 소비 제한·사용자 선택 → 지킬 돈+목표+보상 세팅 → 소비/지킴 트랜잭션 인식
- **참았음 등록 플로우 폐기**, 마이데이터 기반 자동 감지 + 소비 시 항목 최종 확인 방식으로 확정
- **알파 v0.6, 다음 주 중반** 목표로 확정
- 마이데이터는 **더미(페르소나 5개) + JSON 파이프라인** 방식으로 확정
- **3개 에이전트로 병렬 개발** 분배 확정 (이상현·지윤정·이가원)
