## 0. 한 문장 솔루션 정의

> **감정·스트레스·보상 심리로 충동 소비를 하고, 결제가 쉬워지며 소비 순간의 지출 규모를 인지하기 어려운 유저**를 위해, **소비 전·소비 순간에 "충동을 참았다"는 사실을 기록**하고 이를 **즉시 명명된 목표(맥북·도쿄 여행)의 진척으로 전환**해 보여줌으로써, **보상 심리를 억누르는 대신 '획득'의 방향으로 작동**시킨다.
>
> 기록의 마찰이 크므로, **마이데이터 결제 패턴에서 비지출을 감지 → "참은 걸로 기록할까요? 예/아니오"**로 물어 사용자가 스스로 확정하게 하는 방식으로 시작한다. (단, 규제 정합성은 아래 5장 참조 — MVP는 Wizard-of-Oz + 직접 입력으로 라이선스 없이 검증.)

## 1. 여기까지 온 과정 (발산 → 진단 → 수렴 → 선택)

| 단계 | 내용 |
|---|---|
| **발산** | 소비 문제 아이디어 8개 (충동 소비, 소비 규모 비인지, 절약 감각 부재, 잔소리 거부감 등) |
| **진단** | 한 번에 수렴 못 한 이유 4가지 — ① 문제의 타임라인이 다름 ② 성격(인지·심리·시스템)이 다름 ③ 타깃 UX(통제 vs 획득)가 상충 ④ 원인·결과가 섞임 |
| **수렴** | 소비 타임라인을 축으로 3개 그룹화 — A(소비 전·순간: 트리거&결제) / B(직후·정산: 인지&평가) / C(절약·피드백: 보상&거부감) |
| **선택** | **Group A**를 1차 타깃으로 선택. B는 토스·뱅크샐러드가 점령한 레드오션 → 우리는 **A(충동 트리거)를 C(보상 루프)와 이어붙인 빈 공간**을 공략 |

> 관련 아티팩트: `쌍토끼클럽 · 문제 정의 과정` (발산→진단→수렴→선택)

## 2. 핵심 메커니즘 : 참았음 → 획득 프레이밍

**3-STEP 루프**

1. **트리거** — 충동을 참음 (스트레스·보상 심리로 지를 뻔한 배달·쇼핑을 참은 순간)
2. **번역** — 명명된 목표로 전환 ("맥북 사기", "도쿄 여행" — 막연한 자산이 아니라 이름 붙은 가치 소비)
3. **보상** — 진척 프레이밍 ("도쿄 여행 68% 도달!" — 참을수록 목표에 가까워지는 획득 감각)

**설계 원칙 (반드시 지킬 것)**

- **① 진척의 연료 = 참은 순간.** %를 채우는 게 '일반 잔액'이면 그냥 저금통 → 토스에 짐. "참음 → 즉시 진척"의 인과가 붙어야 wedge가 산다.
- **② 진척은 오르기만.** 못 참았다고 68%→65%로 깎으면 손실 프레임(P7 위반). 멈출 순 있어도 뺏기진 않는 비대칭 설계.
- **③ 참음 → 가치 소비 전환.** 목표(맥북·여행)는 결국 더 큰 소비. "충동 습관 소비 → 계획된 가치 소비"로 서사화 (P3 부활).
- **④ 확인 탭 한 번은 절대 없애지 않는다.** 자동화가 진행돼도 "참은 걸로 기록할까요? 예/아니오"의 확인 한 번은 남긴다 — 성취감 앵커 + 귀인 오류 교정을 동시에 담당.
- **⑤ 감정 엔진.** 참음을 "도쿄 여행 +3%"로 보여주는 것 자체가 '샀다는 느낌(획득감)'으로 충동을 달래는 장치.

## 3. "참았음"을 어떻게 감지할 것인가

절약은 "일어나지 않은 소비"라 데이터에 흔적이 없다 (마이데이터는 '쓴 것'만 기록). 그래서 신호를 만드는 두 방식을 비교했고, **양자택일이 아니라 시간축(성숙도 곡선)**으로 결론.

| 방식 | 강점 | 약점 |
|---|---|---|
| **직접 선언** (사람이 센서) | 성취감·능동성 강, 기술 가벼움, 규제 거의 없음, 획득 프레이밍과 직결 | 마찰 높음(매번 기록·누락), 과장 가능 |
| **자동 추정** (데이터가 센서) | 마찰 0, 이탈 방어 | 성취감 약, 귀인 오류(진짜 참음 vs 살 일 없음 구분 불가), 데이터·기술 무거움, 규제 경계 |

### 결론 : 성숙도 곡선 (수동 콜드스타트 → 자동 졸업)

- **Stage 1 · 콜드스타트** — 수동 선언/답장. 목적: 욕구 검증 + 라벨 데이터 수집.
- **Stage 2 · 감지+확인** — 마이데이터 비지출 감지 → "참은 걸로 기록할까요? 예/아니오". 마찰↓, 추론이 사용자를 대신 불러냄.
- **Stage 3 · 자동 우세** — 학습된 패턴으로 감지가 대부분 처리, 사용자는 가끔 확인만. 거의 제로 마찰 리텐션.

**마찰 죽이는 방법** (실행 설계): "기록"이 아니라 "답장"(소비 패턴 분석 타이밍) · 원터치/제로타이핑 · **위시리스트→"안 삼" 전환(유력)** · 홈 위젯(평균 금액) · 다음날 아침 리커버리 · 누락=무벌점.
**스윗스팟 주의**: 마찰을 너무 없애면 자동 추정으로 미끄러져 성취감이 증발. "의도적인 한 번의 선택"으로 느껴질 만큼의 마찰은 남긴다.

> 관련 아티팩트: `쌍토끼클럽 · "참았음" 감지 방식 비교`

## 4. MVP 검증 방법 (중요)

제품 비전은 **감지+확인(Stage 2)**으로 잡되, **검증 단계에서는 비지출 감지를 실제로 만들지 않는다.**

- **이유**: 비지출 감지는 가장 무거운 조각. 마이데이터 사후성 + '사건의 부재' 감지 + 사용자별 baseline 필요 + 귀인 문제. 7/20~8/30 일정에 실제 구현+검증은 비현실적.
- **대안 — Wizard-of-Oz(가짜 감지)**: 감지 알림을 사람이 수동으로 흉내 내서 "참은 걸로 기록할까요?" 경험만 시험. 리스크 큰 엔지니어링 없이 감지+확인 UX 검증.
- **순수 수동 병행**: 같은 프로토타입에 수동 선언 버튼도 한 조각 넣어, "물어봤을 때 반응"과 "스스로 올 의향(욕구)"을 함께 관찰.

### 검증할 핵심 믿음 / 급소

1. **참았음 감지** — 사람들은 참은 순간을 어떻게 기억/기록하나? (스스로 올 의향 vs 물어야 반응) 절약은 흔적이 없다.
2. **진척이 동기인가 부담인가** — "68% 도달"이 뿌듯함인가 죄책감인가? 초반 3% 구간의 지루함(goal-gradient 콜드스타트)은?
3. **실제 이체 없이 감각이 나나** — 진짜 돈을 옮기면 라이선스 영역. '시각 번역'만으로 자산 감각이 충분한지, 아니면 P5 리바운드("티가 안 나서 포기") 재현인지.
4. **감정 엔진** — '샀다는 느낌'으로 충동이 꺼지는가(대체) vs 미뤄질 뿐인가(지연)?

## 5. 경계선 (지킬 것 / 피할 것)

- **규제 라인**: 실제 이체 없는 '시각 번역'까지로 유지. 자동 적립·이체로 넘어가면 라이선스 영역.
- **데이터 수집 규제 정합성 (99_규제_규칙카드 대조)**: 규제 카드상 MVP 채택 방식은 **사용자 직접 입력·업로드**(라이선스 불요)이고, **은행·카드 자동 수집(마이데이터)은 라이선스 필요**. 따라서 "마이데이터 비지출 감지"는 **라이선스 이후 단계(Stage 2~3)**의 비전이며, **MVP는 Wizard-of-Oz(가짜 감지) + 직접 입력**으로 규제 안전지대 안에서 검증한다. → 이 정합성이 "수동으로 시작" 결정을 더 강화.
- **최강 안티패턴**: 잔소리·감시 톤. 답장/알림은 타이밍이 정교해질수록 '감시당한다' 위험이 커진다 → 빈도·톤이 생명, 드물게·축하 톤으로만.
- **저금통화 금지**: 진척의 연료가 '참은 순간'이 아니면 여느 저금통·가계부와 구분 소멸.
