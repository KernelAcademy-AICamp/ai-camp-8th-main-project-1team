## 1. 기회 가설

> **AI가 과거 소비를 분석해 이번 달 줄여볼 카테고리와 근거를 제안하고, 사용자가 고른 절약 강도만큼을 지킬 돈으로 바꿔 한도를 넘긴 만큼 깎이는 모습을 보여주면, 사용자는 절약을 통제가 아니라 자기 선택으로 느끼고 월말 이전에 소비 행동을 조정할 것이다.**

기존 금융 서비스는 이미 발생한 거래를 사후 분석한다. MOA는 과거 데이터를 앞으로의 행동 기준으로 바꾼다.

```text
기존 서비스
거래 발생 → 분류·리포트 → 과소비 인지 → 사용자가 행동 결정

MOA
과거 소비 분석
→ 줄여볼 카테고리 제안
→ 사용자 선택
→ 절약 강도 선택
→ 사용 한도와 지킬 돈 확정
→ 한도를 넘긴 만큼 지킨 돈이 깎임
→ 월말 지킨 돈 확정
→ 캐릭터 누적 성장
```

## 2. 문제 정의와 타깃

### 최종 문제 정의

**감정과 보상 심리로 충동 소비를 반복하는 사용자는, 무엇을 줄여야 할지 판단하지 못하고 절약이 이번 달에 얼마나 지켜지고 있는지 월말 이전에 체감하지 못해 절약을 오래 이어가기 어렵다.**

### 핵심 문제 세트

```text
감정·보상 심리로 소비한다 (P4)
→ 소비 규모를 뒤늦게 인지한다 (P1)
→ 무엇을 줄여야 할지 판단하지 못한다 (P2)
→ 절약 목표의 중간 성과를 체감하지 못한다 (P5)
```

### 대표 타깃

- 스트레스가 쌓이면 배달·쇼핑·택시를 사용한다.
- 카드 결제일에 총액을 보고 후회한다.
- 가계부와 무지출 챌린지를 시도했지만 오래 유지하지 못했다.
- 금융 앱의 분류 리포트를 봐도 다음 행동을 정하지 못한다.
- 절약 자체를 포기한 것은 아니지만 잔소리와 통제를 거부한다.
- 아낀 돈의 사용처는 스스로 정하고 싶어 하며, 서비스가 목표를 강요하는 것을 원하지 않는다.

### 무지출 챌린지와의 차이

MOA는 전체 소비의 무지출을 강요하지 않는다.

- AI는 후보와 근거만 제안한다.
- 사용자가 지켜볼 1~2개 카테고리를 직접 선택한다.
- **얼마나 줄일지도 사용자가 정한다.** 전부 끊을 필요가 없다.
- 선택하지 않은 소비는 평가하지 않는다.
- 한도를 넘겨도 경험을 종료하지 않는다.
- 남은 금액을 부분 달성으로 인정하고 캐릭터의 여정은 계속 이어진다.

## 3. 핵심 설계 결정

### 3.1 직접 기록을 사용하지 않는다

초기에는 사용자가 참은 순간을 직접 기록하는 방식을 검토했다.

> 오늘 배달 22,000원 참았어요

기각 이유:

- 기록이 숙제가 된다.
- 앱을 열지 않으면 데이터가 비어 있다.
- 참은 금액이 주관적이고 과장될 수 있다.
- 실제 소비가 절약을 얼마나 위협하는지 보여주지 못한다.

따라서 사용자가 기록하는 대신 신규 거래를 자동으로 반영한다. 같은 이유로 **참기 세션·자기 보고·사후 금액 입력은 채택하지 않는다.** 자동 거래 원장과 중복되고 기록 부담·과장 입력을 만들기 때문이다.

### 3.2 전체 절감에서 부분 절감으로 되돌린 이유

한때 카테고리를 통째로 끊는 전체 절감 모델을 1차로 확정했었다. 숫자가 하나로 줄어 설명은 쉬웠지만, 화면 시안을 그려 검토하면서 두 가지 문제가 드러났다.

**첫째, 목표가 전부 아니면 전무가 된다.** 택시를 끊기로 하면 이번 달 택시를 한 번도 타지 않아야 완전 달성이다. 출근길에 한 번 늦잠을 자면 그 달의 완전 달성은 사라지고, 남은 3주가 무의미해진다.

**둘째, 게이지가 사용자에게 여유를 준다.** 기준 소비 230,000원에서 시작해 거래마다 깎이는 화면은 `아직 211,500원이나 남았다`로 읽힌다. 절약을 유도하려던 화면이 소비를 허락하는 화면이 됐다.

과거에 부분 절감을 보류했던 이유는 화면에 금액이 네 개 등장한다는 복잡도였다. 그 우려는 실재하지만 **금액 계산이 아니라 화면 설계의 문제**였다. 계산식을 다시 세우면 숫자를 늘리지 않고도 부분 절감을 구현할 수 있다.

### 3.3 현재 모델 — 부분 절감 (전체 절감을 포함한다)

> **사용 한도 = 기준 소비 × (1 − 절약 강도)** — 이번 달 써도 되는 돈
> **지킬 돈 = 기준 소비 × 절약 강도** — 이번 달 지키기로 한 돈

```text
기준 소비 = 카테고리별 최근 3개월 월평균 (잠정)
절약 강도 = 사용자가 카테고리별로 선택
사용 한도 = 기준 소비 × (1 − 절약 강도)
지킬 돈   = 기준 소비 × 절약 강도
누적 소비 = 이번 달 선택 카테고리 실제 지출 합계
남은 한도 = max(사용 한도 − 누적 소비, 0)

지킨 돈   = 지킬 돈 − max(누적 소비 − 사용 한도, 0)
초과 지출 = max(누적 소비 − 기준 소비, 0)
```

**차감 규칙은 하나뿐이다. 한도를 넘긴 만큼만 깎인다.** 깎인 금액과 한도를 넘긴 금액은 언제나 같다.

절약 강도 100%를 고르면 사용 한도가 0원이 되어 카테고리 전체 절감과 완전히 같아진다. **전체 절감은 부분 절감의 특수한 경우다.** 두 모델 중 하나를 고른 것이 아니라 하나가 다른 하나를 품는다.

식을 구간으로 나누면 관계가 드러난다.

```text
누적 소비 ≤ 사용 한도   →   지킨 돈 = 지킬 돈            변하지 않는다
누적 소비 > 사용 한도   →   지킨 돈 = 기준 소비 − 누적 소비
```

두 구간은 사용 한도 지점에서 끊김 없이 이어진다.

### 예시

```text
택시 기준 소비  80,000원 · 강도 50% → 사용 한도 40,000원 · 지킬 돈 40,000원
배달 기준 소비 150,000원 · 강도 50% → 사용 한도 75,000원 · 지킬 돈 75,000원

이번 달 사용 한도 115,000원 · 지킬 돈 115,000원

 8일차  택시 18,500원 결제 · 누적 18,500원 (한도 이내)
        → 지킨 돈 115,000원 · 변화 없음

20일차  누적 130,000원 · 한도 15,000원 초과
        → 지킨 돈 100,000원

월말    택시 42,000원 + 배달 98,000원 · 누적 140,000원 · 초과 25,000원
        → 지킨 돈 90,000원 → 부분 달성
```

### 월말 판정

| 상태 | 조건 | 의미 |
|---|---|---|
| 완전 달성 | 누적 소비 ≤ 사용 한도 | 한도를 지켜 지킬 돈을 전부 지킴 |
| 부분 달성 | 사용 한도 < 누적 소비 < 기준 소비 | 한도는 넘겼지만 평소보다 덜 써서 일부를 지킴 |
| 미달성 | 누적 소비 ≥ 기준 소비 | 평소 수준 이상 사용해 지킨 돈이 0원 |

월중에는 결과가 확정되지 않으므로 `지키는 중` 또는 `지킬 돈 소진`으로만 표시한다. 완전/부분/미달성은 내부 분석용이며, 사용자 화면에는 지킨 금액과 성장을 중심으로 보여주고 실패 낙인을 쓰지 않는다.

### 3.4 증가하는 것은 캐릭터의 성장이다

현재 모델에서 지킨 돈은 월초에 확정되고 이후 유지되거나 깎인다. **월중에 늘어나는 금액은 존재하지 않는다.** 사용자에게 쌓이는 경험을 주려면 금액이 아닌 축이 필요하다.

MOA는 이 축을 별도의 목표(물건)나 육성 게임이 아니라 **캐릭터의 성장**으로 푼다. 사용자의 목표는 `아낄 수 있는 돈을 아끼는 행위` 자체이며, 이를 동기 치환해 지키기 위해 여정을 이어가는 캐릭터가 성장하는 모습으로 표현한다. 진행 구조는 두 시간축이다.

| 층 | 사용자 질문 | 연료 | 월 전환 |
|---|---|---|---|
| 월간 금액 (월 게이지) | 이번 달 얼마를 지키는 중인가? | 지킬 돈 대비 지킨 돈. 한도 초과분만큼 깎임 | 새 챕터로 시작 |
| 누적 성장 (성장 레벨) | 캐릭터와 얼마나 오래 지켜왔는가? | 월말 확정 지킨 돈이 누적 | 유지 · 리셋·퇴화 없음 |

`돈을 아낀 만큼(월 게이지는 깎일 수 있어도) 캐릭터는 계속 성장한다`는 두 축의 관계가 핵심 경험이다. 외부 목적지와 종착점은 없다. 매월 한 챕터를 지나 다음 챕터로 이어지는 열린 여정이다.

**폐기한 장치.** 여행·물건 목표와 목적지, 예상 페이스·도착률, 보상·금융상품 추천, 육성 오브젝트, 보호 쿠폰, 지킨 날 원장, 참기 세션, OS 위젯은 모두 제거했다. 절약 서비스가 새로운 소비 동기를 심는 모순을 피하고, 감정 이입 대상을 캐릭터 하나로 유지하기 위해서다.

## 4. 핵심 사용자 경험

```text
[최초 1회]
더미 마이데이터 연결 (카드 + 계좌)
→ 소비 분석 → 카테고리·강도 선택 → 지킬 돈 확정
→ 캐릭터 이름 설정 · 첫 챕터 시작

[월초]
지난달 결과 확인
→ AI 소비 분석
→ 줄여볼 카테고리와 근거 확인
→ 사용자 카테고리 선택
→ 카테고리별 절약 강도 선택
→ 사용 한도와 지킬 돈 확정 · 새 챕터 시작

[월중]
Home 확인
↔ 신규 거래 시뮬레이션
→ 선택 카테고리면 누적 소비·남은 한도 갱신
→ 한도를 넘었으면 초과분만큼 지킨 돈 차감
→ 알림·맥락 태그·오분류 수정
→ 캐릭터는 이번 달의 길을 지키는 모습만 표현 (성장은 아직 반영 안 함)

[월말]
지킨 돈 확정
→ 완전 달성·부분 달성·미달성 내부 판정
→ 월말 리포트
→ 월말 확정 지킨 돈을 누적 성장에 1회 반영 · 성장 연출
→ 다음 달 카테고리·강도 선택
```

**누적 성장만 이 루프 밖에 있다.** 월이 바뀌어도 리셋되지 않고 계속 누적된다.

### 거래 처리

- 선택 카테고리 거래는 자동으로 누적 소비에 반영한다.
- 미선택 카테고리 거래는 지킬 돈 계산에 영향을 주지 않는다.
- 계좌 거래는 누적 소비에 반영하지 않는다. 카드 대금 출금과 이중 계상되기 때문이다.
- 분류 신뢰도가 낮은 거래만 사용자에게 카테고리 확인을 요청한다.
- 오분류 수정과 거래 취소·환불만 금액을 복구한다.
- 맥락 태그는 금액을 바꾸지 않는다.

### 소비 맥락 태그

```text
🎯 계획했던 소비
😅 줄이려던 소비
🚨 불가피한 소비
```

불가피한 소비가 반복되면 해당 카테고리를 다음 달 후보에서 제외하거나 강도를 낮춰 제안한다.

## 5. Home 화면

Home은 실행 후 3초 안에 이번 달 지킴 상태를 이해하게 하는 핵심 화면이다.

### 한 숫자 구조

Home의 금액 주 지표는 `현재 지키는 금액` 하나다. 기준 소비·사용 한도·남은 한도·소진률·목표선은 노출하지 않는다.

| 구분 | 처리 |
|---|---|
| 사용자 노출 | 현재 지키는 금액 |
| 내부 계산 | 기준 소비 · 사용 한도 · 누적 소비 · 남은 한도 · 초과액 |

**목표선은 표시하지 않는다.** 지킨 돈이 깎이기 시작하는 순간 자체가 한도 초과 신호이므로 별도의 선이 필요 없다. 사용 한도 안에서는 금액이 그대로 유지되고 서비스가 침묵한다.

### 표시 정보

- 현재 지키는 금액 (주 지표)
- 캐릭터 여정 영역 (캐릭터 · 이름 · 성장 레벨 · 이번 달 챕터)
- 선택 카테고리와 절약 강도 (금액 한도는 표시하지 않음)
- 월중 상태(`지키는 중` / `지킬 돈 소진`)
- 지킴이 말풍선 (차감 설명 · 회복 안내)
- 최근 거래와 맥락 태그
- 초과 지출 (지킬 돈 소진 후)

누적 지킨 돈은 Home에 두지 않고 캐릭터 성장 상세에서 확인한다. 현재 지키는 금액(월)과 장기 누적 금액을 같은 위계에 놓지 않기 위해서다.

### 지킨 돈 동작

```text
 1일   누적      0원 (한도 이내)      지킨 돈 115,000원 지키는 중
 8일   누적 18,500원 (한도 이내)      지킨 돈 115,000원 지키는 중 · 알림 없음
20일   누적 130,000원 (15,000 초과)   지킨 돈 100,000원 지키는 중 · 차감 알림
월말   누적 140,000원 (25,000 초과)   지킨 돈  90,000원 지킴
```

- 시작값은 이번 달 지킬 돈과 같다.
- 사용 한도 안에서는 변하지 않는다.
- 한도를 넘어선 뒤부터 초과분만큼 줄어들며 0원에서 멈춘다.

월중 금액은 아직 계좌에 들어간 돈이 아니므로 `모았다`고 표현하지 않고 `지키는 중`으로 쓴다. 캐릭터의 성장은 금액이 아니므로 이 제약을 받지 않지만, 월중에는 성장을 확정하지 않는다.

## 6. 지속 동기 — 캐릭터 여정과 성장

### 지킴이 캐릭터

지킴이는 세 역할을 갖는다.

1. 사용자가 지키기로 한 돈을 함께 지키는 동행자
2. ②가 생성하는 알림·회고 문장을 비난 없이 말하는 화자
3. 월말 확정 지킨 돈이 쌓일수록 성장하는 유일한 감정 이입 대상

캐릭터는 금액을 계산하거나 사용자의 소비를 판정하지 않는다. 지출에 슬퍼하거나 약해지지 않고, 월말 지킨 돈이 0원이어도 레벨과 외형을 유지한다.

```text
"정해둔 범위를 15,000원 넘었어요. 지금 100,000원 지키는 중이에요."
"금액이 줄었지만 아직 100,000원이 남아 있어요. 남은 열흘이 더 중요해요."
```

### 성장 규칙

성장은 월말 확정 후 한 번만 발생한다. 월중 금액은 취소·환불·재분류로 바뀔 수 있어 즉시 반영하면 되돌려야 하므로, 누적형 원칙과 충돌한다.

```text
이번 달 성장 반영액 = 월말 확정 지킨 돈
누적 지킨 돈       = 이전 누적 지킨 돈 + 이번 달 성장 반영액
성장 레벨          = 누적 지킨 돈이 속한 구간
```

- 누적 지킨 돈과 성장 레벨은 감소하거나 리셋되지 않는다.
- 같은 사용자의 같은 월은 멱등키로 한 번만 반영한다.
- 0원을 지킨 달에는 성장이 멈출 뿐 퇴화·비난하지 않는다.
- 성장 단계는 결제·과금으로 구매할 수 없고, 성장이 소비를 유도하는 아이템·상점 구조를 두지 않는다.
- 성장 레벨 구간과 외형 단계 수는 미정이다.

### 월간 챕터

매월은 종착점 없는 여정의 새 구간이다.

```text
월초   새 챕터 시작 · 이번 달 카테고리와 강도 확정
월중   캐릭터가 현재 지킬 돈을 지키는 상태
월말   최종 지킨 돈 확정 · 누적 성장 · 챕터 기록 저장
다음 달 새 챕터 시작 · 누적 레벨·외형 유지
```

챕터는 `완주/실패`로 낙인찍지 않는다. 사용자 메시지는 `이번 달 {금액}원을 지켰어요`와 성장 결과를 중심으로 구성한다.

### 소비 목표·보상 비개입, 금융상품 정보성 추천

- 여행·물건·구매 목표를 추천하거나 입력받지 않는다.
- 목적지·목표 금액·도착률·남은 거리·예상 페이스를 표시하지 않는다.
- 지킨 돈으로 살 소비형 보상·상점 아이템을 추천하지 않는다.
- 지킨 돈의 실제 사용처는 사용자의 자유이며 서비스가 묻거나 평가하지 않는다.
- **예외:** 아낀 돈의 보관처(저축) 관점의 금융상품 **정보성 비교 추천**은 ③이 제공한다. 실제 상품 정보를 소비 패턴·취향에 맞춰 개인화하되, 판매·중개·가입 대행 없이 `가입은 각 사로` 안내하고 목표·페이스로 제시하지 않는다.

## 7. 시스템 아키텍처 — 3개 Agent

```text
더미 마이데이터 서버 (카드 + 계좌)
        │
        ▼
① 소비 분석 Agent
   ├─ 기준 소비·절약 후보·권장 강도·근거 ──▶ ② 지킴·성장 Agent
   └─ 소비 분석 요약(성향·충동지수 포함) ───▶ ③ 취향·추천 Agent
                                              ▲
② 지킴·성장 Agent ── 지킴·성장 상태 ──────────┘
외부 실제 금융상품 정보 ──▶ ③
① ◀── 월말 지킴 실적 ── ②
③ ──▶ 사용자: 주간·월말 리포트 + 맞춤 금융상품 정보성 추천
```

### ① 소비 분석 Agent — 이상현

- 입력: 카드·계좌 거래 리스트 · 전월 지킴 실적 · 맥락 태그 · 오분류 이력
- 출력: 기준 소비 · 줄여볼 카테고리 · 권장 강도 · 추천 근거 · 신뢰도 · 반복 결제 · 고정지출 · 시간 패턴 · 가치 소비 보호 신호 · 성향 프로필
- LLM: 애매한 거래 재분류 · 추천 근거 · 성향 문장
- 규칙: 집계 · 평균 · 반복 결제 · 가맹점명 키워드 분류 · 계좌 적요 규칙 분류 · 기준 소비

### ② 지킴·성장 Agent — 지윤정

- 입력: 분석 결과 · 사용자 선택 카테고리와 강도 · 신규 거래 · 재분류 · 캐릭터 이름
- 출력: 사용 한도 · 지킬 돈 · 누적 소비 · 남은 한도 · 지킨 돈 · 월중 상태 · 실시간 알림 · 월말 확정 지킨 돈 · 캐릭터·레벨·외형·월간 챕터·누적 지킨 돈(성장 상태)
- LLM: 지킴이 캐릭터의 비난 없는 실시간 거래 알림 문구
- 규칙: 카테고리 매칭 · 한도 초과분 차감 · 월중 상태 판정 · 월말 지킨 돈 확정 · 월말 확정분 성장 누적(멱등)·성장 구간 판정·챕터 전이

### ③ 취향·추천 Agent — 이가원

- 입력: 마이데이터(취향 분석용) · ①의 소비 분석 요약(성향 프로필·충동구매지수 포함) · ②의 지킴·성장 상태 · 외부 실제 금융상품 정보
- 출력: 취향·성향 분석 · 주간 리포트 · 월말 리포트 · 맞춤 금융상품 정보성 비교 추천
- 주기: 주간(요약) · 월말(리포트·추천)
- LLM: 리포트·성장·회고 서술 · 추천 사유 문장
- 규칙: 리포트 집계 · 취향 특징 추출 · 상품 매칭·정렬(방식 미정)
- 제약: 금융상품은 정보성 비교만(판매·중개·가입 대행 없음, `가입은 각 사로`), 목표·페이스 재도입 금지

### 책임 원칙

- AI는 후보와 근거를 제안한다.
- 사용자가 카테고리와 강도를 확정한다.
- 금액 계산과 상태 판정은 결정론적 규칙 엔진이 수행한다.
- LLM은 금액·레벨을 계산하거나 변경하지 않는다. 지킴이 캐릭터는 문장의 화자일 뿐 판정 주체가 아니다.
- 취향·추천 Agent는 ①의 선택·강도나 ②의 금액·성장 값을 직접 바꾸지 않는다. 금융상품은 정보성 비교만 제공한다.

Agent 간 연결(A·B·C·D·F·G)의 스키마와 쓰기 권한은 `03_2차자료/07_에이전트_연결부.md`를 정본으로 한다. 성장은 ② 내부에서 처리하며(옛 ②→③ 성장 반영·③→① 요약 폐기), 월말에는 ②가 지킴 실적(G)을 발행하고 ①은 이를 받은 뒤 다음 달 분석을 시작한다. ③의 리포트·추천은 사용자 산출물이다.

## 8. 마이데이터와 거래 분류

### 더미 마이데이터

- 실제 마이데이터 사업자와 연결하지 않는다.
- 카드 국내 승인내역과 은행 수시입출금 거래내역 표준 스키마를 모사한다. 보험·증권은 수집하지 않는다.
- 최근 3개월 거래를 조회한다.
- 가맹점명·사업자등록번호 선택 동의가 없으면 분석 진행을 차단한다.
- 페이지네이션·기관별 부분 실패·취소·정정을 구현한다.
- `/demo/*`에 시계 이동·거래 주입·초기화 기능을 둔다.
- 고정 시드로 발표 시나리오를 재현한다.

마이데이터는 정보제공자가 결제 내역을 push하는 구조가 아니다. 1차 구현은 더미 거래를 실시간처럼 주입하지만 문서·발표에서는 `실시간 결제 알림`으로 표현하지 않는다.

### 계좌는 왜 받는가

> **누적 소비는 카드 승인내역만으로 계산한다.**

카드 결제 대금은 나중에 계좌에서 빠져나간다. 계좌 출금을 소비로 세면 같은 지출을 두 번 세게 되므로, 계좌 거래는 금액 계산에서 제외한다.

계좌 거래는 다음 용도로만 쓴다.

- 고정지출 탐지 — 월세·통신비·보험료를 절약 후보에서 제외
- 구독 탐지 — 같은 금액이 같은 날 반복되는 자동이체
- 가치 소비 보호 신호 — 반복되는 취미·가족·건강 소비를 절약 후보에서 보호
- 절약 여력 산출 — 급여 입금과 고정지출의 차이

계좌 거래에는 가맹점명이 없고 적요만 있어 가맹점명 분류 모델을 그대로 쓸 수 없다. 별도의 규칙 기반 분류를 사용한다.

### 카테고리 분류

마이데이터 카드 승인내역에는 MCC가 없으므로 가맹점명 기반으로 분류한다.

```text
1차 규칙: 가맹점명 키워드 사전
→ 미분류
2차 ML: 가맹점명 텍스트 분류
→ 신뢰도 미달
3차 LLM: 분류 + 불확실 표시
→ 사용자 수정
```

- 상위 단계에서 확정되면 하위 단계를 호출하지 않는다.
- 오분류 수정은 키워드 사전·학습 데이터에 반영한다.
- 자체 더미만 학습해 생성 규칙을 재학습하는 순환 구조를 피한다.

- 가상 출금 계좌·지킴 계좌 개설
- 월말 지킨 돈 가상 이체
- 지킴 계좌 누적 잔액
- 중복 이체 방지와 가상 거래내역

월말 지킨 돈이 0원보다 크면 완전 달성 여부와 관계없이 가상 지킴 계좌로 옮긴다.

### Later

- 월중 강도 변경
- 성역 카테고리 명시 지정
- 소비 행동 단위 절약
- OS 위젯
- 참기 세션·자기 보고
- 실제 마이데이터 사업자 연동
- 실제 계좌 개설·이체
- 사업자등록번호 기반 업종 역조회
- 미선택 카테고리 풍선효과 분석
- 푸시 알림
- 커뮤니티·공동 챌린지
- AI 개인화 고도화

> 월말 리포트를 Must로 올렸다. 리포트의 `다음 달 정하기`가 월초 화면으로 돌아가는 유일한 경로이므로, 리포트가 없으면 월간 루프가 닫히지 않는다.

> **리포트는 주간과 월간을 모두 준다.** 한 달은 피드백 주기로 길어 월말 리포트만 두면 화면이 29일 동안 비어 있다. 다만 **주간은 요약이고 월간은 확정이다.** 주간 리포트는 달성 여부를 판정하지 않고 성장도 확정하지 않으므로 월간 루프와 계산 모델이 흔들리지 않는다. 주간 집계는 ③이 이미 받고 있는 (C) 월중 표시 상태를 주 단위로 묶어 만들며 Agent 연결부에 새 연결을 추가하지 않는다.
>
> **성역 설정은 Later로 뺐다.** 현재 모델에서는 선택하지 않은 카테고리가 곧 성역이다. 미선택 카테고리는 계산 대상이 아니고 알림도 침묵하므로 기능이 이미 구조에 내장돼 있다.

## 10. 가상 지킴 계좌

가상 계좌 기능은 마이데이터 조회 API와 분리한다.

```text
더미 마이데이터 서버 → 카드 거래 · 계좌 거래 제공
MOA Agent 시스템      → 지킬 돈 추적·월말 지킨 돈 확정
가상 금융 서버        → 계좌 개설·잔액·가상 이체·누적 원장
```

| 용도 | 엔드포인트 |
|---|---|
| 계좌 개설 | `POST /demo/banking/accounts` |
| 계좌·잔액 조회 | `GET /demo/banking/accounts/{account_id}` |
| 거래내역 조회 | `GET /demo/banking/accounts/{account_id}/transactions` |
| 가상 이체 | `POST /demo/banking/transfers` |

- 이체액은 ②(지킴·성장 Agent)의 규칙 엔진이 확정한 지킨 돈과 같아야 한다.
- 0원은 이체하지 않는다.
- 사용자·월별 멱등키로 중복 이체를 막는다.
- 계좌에는 누적 잔액과 월별 이체 내역만 표시하며 목표 진척률이나 사용처를 연결하지 않는다.
- 실제 계좌와 실제 자금은 사용하지 않는다.

## 11. 성공 기준

### 제품·UX

| 지표 | 기준 |
|---|---:|
| 카테고리 1개 이상 선택 | 필수 |
| AI 추천 카테고리 수용률 | 측정 |
| AI 권장 강도 수용률 | 측정 |
| `사용 한도`와 `지킬 돈` 구분 이해도 | 80% 이상 |
| 차감 이유 설명 가능 응답 | 80% 이상 |
| AI 추천 신뢰도 | 80% 이상 |
| 월간 한 사이클 실행 | 가능 |
| 완전·부분·미달성 비율과 평균 지킨 돈 | 측정 |
| 월말 성장 확인률 · 다음 달 챕터 시작률 | 측정 |

리텐션은 핵심 질문으로 유지하되 더미 데이터 기반 MVP에서의 측정 방법은 별도 설계한다. 월말 성장 확인률과 다음 달 챕터 시작률이 대리 지표가 될 수 있는지 함께 검토한다.

### 시스템

- 선택 카테고리 거래만 누적 소비에 반영된다.
- 계좌 거래는 누적 소비에 반영되지 않는다.
- 누적 소비가 사용 한도 이내이면 지킨 돈이 변하지 않는다.
- 누적 소비가 사용 한도를 넘으면 초과분만큼만 지킨 돈이 줄어든다.
- 절약 강도 100%를 선택하면 사용 한도가 0원이 되고 첫 거래부터 지킨 돈이 줄어든다.
- 맥락 태그는 금액을 바꾸지 않는다.
- 오분류와 취소·환불은 금액을 복구한다.
- 지킨 돈은 0원에서 멈추고 초과 지출을 별도 기록한다.
- 월중 지킨 돈의 변화는 누적 성장에 즉시 반영되지 않는다.
- 월말 확정 지킨 돈이 동일 월에 한 번만 성장 원장에 반영된다.
- 누적 지킨 돈과 캐릭터 레벨은 어떤 경우에도 감소하지 않는다.
- 0원을 지킨 달에도 캐릭터가 퇴화하거나 비난하는 문구가 나오지 않는다.
- 목적지·남은 거리·소비형 보상은 어떤 화면에도 나타나지 않는다. 금융상품은 정보성 비교만 제공한다.
- 월말 판정이 완전 달성·부분 달성·미달성 규칙과 일치한다.
- 다음 달 분석은 ②의 월말 지킴 실적(G)을 받은 뒤 시작한다.

## 12. 검증 질문

- 사용자는 `사용 한도`와 `지킬 돈`을 구분해서 이해하는가?
- AI가 제안한 카테고리와 근거를 신뢰하는가?
- 절약 강도 선택이 부담을 줄이는가, 결정을 늘리는가?
- 한도 내에서 지킨 돈이 변하지 않는 것을 안정으로 느끼는가, 무감각으로 느끼는가?
- 한도를 넘긴 순간의 차감이 행동 조정으로 이어지는가?
- 1~2개 카테고리 선택이 현실적인가?
- 부분 달성도 성과로 받아들이는가?
- 소비 맥락 태그가 금액과 무관하다는 점을 이해하는가?
- 외부 소비 목표 없이도 캐릭터의 여정과 성장이 절약 동기로 작동하는가?
- 월중 `지키는 금액`과 장기 `누적 지킨 돈`을 서로 다른 숫자로 이해하는가?
- 월말에만 성장하는 구조를 자연스럽게 받아들이는가?
- 종착점 없는 월간 챕터가 다음 달 재방문 동기가 되는가?
- 더미 데이터 환경에서 리텐션을 어떻게 측정할 것인가?
