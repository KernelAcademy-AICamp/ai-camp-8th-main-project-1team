# 01b. 검증 리포트 보완

> 검증 방식: 웹 검색 → 원문 확인 → 교차 검토  
> 결과: 25개 주장 중 18개 유지, 7개 제외

## 이번 보완에서 달라진 점

### 1. 소비를 늦게 체감하는 이유가 선명해졌다

카드 결제는 소비 시점과 실제 출금 시점이 다르다.

이 시간차가 지출 감각을 약하게 만든다는 이론과 사례를 보강했다.

### 2. 감정 소비의 규모를 뒷받침하는 근거가 늘었다

‘시발비용’, 홧김 소비, 보상 소비가 일부 개인의 특이한 행동이 아니라 널리 사용되는 소비 언어라는 점을 확인했다.

### 3. 보상 소비와 저축 실패가 연결됐다

감정 소비는 한 번의 충동으로 끝나지 않는다.

반복되면 저축 목표를 무너뜨린다.

따라서 우리의 문제는 단순한 소비 기록이 아니라 목표 달성을 방해하는 행동 패턴이다.

### 4. 자동 분석의 신뢰 문제가 더 중요해졌다

분류 오류, 누락, 느린 반영, 수정 피드백 부족이 반복해서 확인됐다.

MVP에서도 사용자가 결과를 수정할 수 있어야 한다.

## 사용하지 않기로 한 주장

아래 주장은 원출처를 확인하지 못했거나 일반화가 과했다.

최종 발표와 PRD의 근거로 사용하지 않는다.

- 직장인 85.5%가 월평균 20만 원을 홧김 소비한다는 수치
- 20~30대 59.1%가 스트레스 소비를 정당화한다는 수치
- 카드 결제일의 놀람을 전체 사용자의 대표 반응으로 보는 주장
- 무지출 챌린지가 주로 금융 앱에서 인증된다는 주장
- 장시간 노동이 저축 실패의 핵심 원인이라는 일반화

## 이번 검증에서 얻은 원칙

1. 통계 수치는 반드시 원출처를 확인한다
2. 개인 사례는 사례로만 사용한다
3. 여러 채널에서 반복되는 발화와 대표 수치를 구분한다
4. 서비스 방향에 유리한 근거만 남기지 않는다
5. 기각한 근거도 문서에 남긴다

## 제품에 추가로 반영할 것

### 오류 수정 3단계

1. 사용자가 오류를 바로 발견한다
2. 한 번에 수정한다
3. 다음 분석에 반영됐음을 확인한다

### 포지셔닝 문장

> 돈을 적게 쓰라고 혼내는 서비스가 아니다.  
> 사용자가 지키고 싶은 돈에 이름을 붙여주는 서비스다.
