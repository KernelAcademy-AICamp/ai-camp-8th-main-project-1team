# 정리 파일 — 가맹점 이름 / 사업자등록번호 / 주소

마이데이터 대량 생성기(`com.finntech.mydata.generation`)가 만든 **고유 가맹점 목록**이다.
사용자에게는 결제에 실린 **사업자등록번호**만 전달되고, 이 번호로 가맹점 주소를 조회한다(번호→주소).

## 파일
- `merchants-part01.csv` ~ `merchants-part08.csv` — 분할본. 각 파트에 헤더 포함.
  - 합본 `merchants.csv`(약 282MB)는 GitHub 파일당 100MB 제한을 넘어 `.gitignore` 처리.
  - 합치기: `cat merchants-part*.csv | awk 'NR==1||FNR>1'` (헤더 1회만).
- 헤더: `가맹점명,사업자등록번호,주소,온라인`

## 규모(2026-07-24 생성분)
- 결제 11,203,712건 / 사용자 4,962명 / **고유 가맹점 3,817,017건**
- seed 20260721, history-days 280, 코호트 2025-12-15~2026-02-15 (재현 가능)
- 신원 키에 전체 행정구역(시도+시군구+동)을 써서 전국 중복 동명('중앙동' 등) 중복배정을 제거했고,
  10자리 번호 공간의 순수 해시충돌로 번호↔주소가 어긋나던 13,443건(+거래 109,194건)은 제거해
  **번호→주소 매핑이 100% 유일**하다(다중주소 번호 0 검증).

## 원리
- 사업자번호·지번주소는 가맹점 **신원(오프라인=이름+동, 온라인=이름)의 결정론 함수** →
  같은 점포는 항상 같은 번호·주소(중복 배정 없음). 사업자번호는 국세청 검증식(XXX-YY-ZZZZA)을 만족.
- 온라인 가맹점은 전국 **본사(HQ)** 고정 주소·번호.
- 이 목록은 결제(`mydata_payment`)에서 `business_number` DISTINCT로 집계해 `mydata_merchant` 테이블·CSV로 낸다.
- 조회: `GET /bank/mydata/merchant/{사업자번호}`(mydata) / `GET /api/mydata/merchant/{사업자번호}`(backend 프록시).
